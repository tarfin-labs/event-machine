## 1. Current State Analysis

### What Works (✓ VERIFIED)

| Capability | How | Limitation |
|------------|-----|------------|
| Fake individual behaviors | `MyAction::fake()` + `::run()` | Only via `::run()`, **NOT** during `Machine::send()` |
| shouldRun / shouldReturn | `MyGuard::shouldReturn(true)` | Same `::run()` limitation |
| assertRan / assertNotRan | Mockery-based verification | Same `::run()` limitation |
| resetFakes / resetAllFakes | Per-class and global cleanup | Works correctly |
| ResolvesBehaviors | `Machine::getGuard('name')` | Only string-keyed, not FQCN references |
| shouldPersist = false | DB-less testing | Works for definition-level tests |
| History inspection | `$state->history->pluck('type')` | Verbose, no helper assertions |

### What Doesn't Work (✓ VERIFIED)

| Gap | Root Cause | Impact |
|-----|-----------|--------|
| **Fakes ignored during `Machine::send()`** | `getInvokableBehavior()` uses `new $class()` | **P0 — Everything else depends on this** |
| **No constructor DI** | `new $class()` bypasses container | **P0 — Forces service locator anti-pattern** |
| **No DI in `::run()`** | `::run()` passes raw args; machine uses reflection DI | **P0 — Can't unit test with realistic params** |
| **No fluent test wrapper** | Tests require manual `send()` + `expect()` chains | **P1 — Boilerplate in every test** |
| **EventBehavior not testable** | No factory, extends Spatie Data | P1 — Hard to construct for tests |
| **No parallel dispatch verification** | No Queue::fake() integration guidance | P2 — Parallel behavior hard to test |

### All Behavior Invocation Points in the Engine

Every location where `getInvokableBehavior()` is called — all must respect fakes:

| Invocation Point | File | Method |
|-----------------|------|--------|
| Guard evaluation | `TransitionDefinition` | `getFirstValidTransitionBranch()` |
| Calculator execution | `TransitionDefinition` | `runCalculators()` |
| Transition actions | `TransitionBranch` | `runActions()` → `MachineDefinition::runAction()` |
| Exit actions | `StateDefinition` | `runExitActions()` → `MachineDefinition::runAction()` |
| Entry actions | `StateDefinition` | `runEntryActions()` → `MachineDefinition::runAction()` |

All flow through `MachineDefinition::getInvokableBehavior()` — fixing that single method fixes all invocation points.

## 2. Breaking Changes & Upgrade Guide

### Breaking Change 1: Behavior Resolution via Container

**Before:**
```php
// MachineDefinition::getInvokableBehavior()
return new $behaviorDefinition($this->eventQueue);
```

**After:**
```php
return App::make($behaviorDefinition, ['eventQueue' => $this->eventQueue]);
```

**Who is affected:**

| Scenario | Impact |
|----------|--------|
| Behaviors with no constructor override | ✅ No change — container passes `$eventQueue` to default constructor |
| Behaviors with constructor DI (new capability) | ✅ Works — container resolves service dependencies automatically |
| Behaviors with non-resolvable required constructor params | ⚠️ **Breaking** — container can't resolve arbitrary scalars without bindings |

**Migration:**

```php
// ❌ BREAKS — container can't resolve `string $prefix` automatically
class BadBehavior extends ActionBehavior
{
    public function __construct(string $prefix, ?Collection $eventQueue = null)
    {
        parent::__construct($eventQueue);
    }
}

// ✅ FIX — use a typed dependency or register a binding
class FixedBehavior extends ActionBehavior
{
    public function __construct(PrefixConfig $config, ?Collection $eventQueue = null)
    {
        parent::__construct($eventQueue);
    }
}
// OR: register in a service provider
$this->app->when(BadBehavior::class)->needs('$prefix')->give('my_prefix');
```

**Note:** The `❌ BREAKS` scenario was already broken with `new $class($eventQueue)` — a Collection would be passed as the `$prefix` string parameter, causing a type mismatch. Container resolution **surfaces existing bugs** rather than creating new ones.

### Breaking Change 2: `InvokableBehavior::run()` Always Uses Container

**Before:**
```php
$instance = static::isFaked() ? App::make(static::class) : new static();
```

**After:**
```php
$instance = App::make(static::class);
```

**Who is affected:** Code that calls `::run()` and relies on `new static()` bypassing the container. In practice, no one — this method was only used in tests.

### Breaking Change 3: `Fakeable::fake()` Uses `App::bind()` with Closure

`Fakeable::fake()` now registers fakes via `App::bind($class, fn() => $mock)` instead of the previous `App::bind()` + `$fakes` array dual storage. The closure always returns the same Mockery mock instance, making it effectively singleton-scoped within a test.

**Why `App::bind()` instead of `App::instance()`:** Laravel's `App::instance()` bindings are **ignored** when `App::make()` is called with explicit parameters (e.g., `App::make(MyGuard::class, ['eventQueue' => $queue])`). Since `getInvokableBehavior()` always passes `['eventQueue' => $this->eventQueue]`, `App::instance()` would silently bypass the fake. `App::bind()` with a closure is called regardless of extra parameters.

**Cleanup change:** `resetFakes()` now uses `app()->offsetUnset($class)` instead of `App::forgetInstance($class)`, because `forgetInstance()` only removes instance bindings, not closure bindings. The public API is identical.

### Upgrade Guide Summary

```markdown
## Upgrading to vX.0

### Behavior Resolution (Breaking)

Behaviors are now resolved through Laravel's service container instead of
direct instantiation. This enables constructor dependency injection.

**Action required if you:**

1. Override InvokableBehavior::__construct() with non-injectable parameters
   (plain string, int, array without defaults):
   → Register a container binding for the parameter.

2. Extend constructors positionally:
   → Use named, typed parameters.

**No action required if you:**
- Use the default InvokableBehavior constructor (most behaviors)
- Only override __invoke() (most behaviors)
- Use typed service dependencies in constructors (new capability!)

### New Testing Features

- Machine::test() — Livewire-style fluent test wrapper
- State::forTesting() — Lightweight state factory for unit tests
- InvokableBehavior::runWithState() — Isolated testing with engine-identical DI
- EventBehavior::factory() — Test factory for event construction
- Constructor DI — Behaviors can now inject service dependencies
```

## 3. Core Change: Container Resolution

### The Change

One method changes — `MachineDefinition::getInvokableBehavior()`:

```php
public function getInvokableBehavior(
    string $behaviorDefinition,
    BehaviorType $behaviorType,
): null|callable|InvokableBehavior {
    // FQCN reference → resolve through container
    if (is_subclass_of($behaviorDefinition, InvokableBehavior::class)) {
        return App::make($behaviorDefinition, ['eventQueue' => $this->eventQueue]);
    }

    // String key → look up in behavior registry
    $invokableBehavior = $this->behavior[$behaviorType->value][$behaviorDefinition] ?? null;

    // Registry returns a class name → resolve through container
    if ($invokableBehavior !== null && !is_callable($invokableBehavior)) {
        return App::make($invokableBehavior, ['eventQueue' => $this->eventQueue]);
    }

    // Registry returns null → behavior not found
    if ($invokableBehavior === null) {
        throw BehaviorNotFoundException::build($behaviorDefinition);
    }

    // Registry returns a callable (closure) → use directly
    return $invokableBehavior;
}
```

### How Container Resolution Works

```
Container resolves: App::make(IsEvenGuard::class, ['eventQueue' => $collection])

1. Is there an App::bind() binding?     → Yes → call closure → return mock
                                        → No → continue

2. Reflective auto-resolution:
   - Reflect constructor(?Collection $eventQueue = null)
   - Match named param 'eventQueue' → $collection from explicit params
   - Resolve any typed service dependencies from container
   - Return new instance with all dependencies injected
```

When `MyGuard::fake()` calls `App::bind(MyGuard::class, fn() => $mock)`:

```
App::make(MyGuard::class, ['eventQueue' => $queue])
  → Bind closure exists → call closure → return $mock
  → eventQueue parameter IGNORED (closure returns pre-built mock)
```

> **Why not `App::instance()`?** Instance bindings are silently bypassed when `App::make()` receives a second `$parameters` argument. Since `getInvokableBehavior()` always passes `['eventQueue' => $this->eventQueue]`, fakes registered via `App::instance()` would never be returned. `App::bind()` closures are always invoked.

### Edge Cases

**Mock + Reflection DI (`injectInvokableBehaviorParameters`):**

After `getInvokableBehavior()` returns a mock, the engine does:

```php
$params = InvokableBehavior::injectInvokableBehaviorParameters(
    actionBehavior: $mock,
    state: $state,
    eventBehavior: $eventBehavior,
);
($mock)(...$params);
```

`Mockery::mock(MyGuard::class)` creates a class **extending** `MyGuard`. Reflection on `__invoke` finds the real class's parameter types → DI injection works correctly → mock's `__invoke` is called with the right parameters → returns preconfigured value. **Works.**

**Mock + `validateRequiredContext`:**

Mock extends real class → `$requiredContext` comes from the real class's static property → validation runs against real requirements. **Tests must set up valid context.** This is the correct default.

**Mock + `$shouldLog`:**

Mockery proxies property access. Unset properties return `null`. `null ?? false` = `false`. **Works.**

**Parallel dispatch — fakes in queue workers:**

| Execution Mode | Fakes Work? | Correct? |
|---------------|-------------|----------|
| Same process (sync queue, `withoutParallelDispatch`) | ✅ Yes | ✅ Yes — test process |
| Separate workers (real queue) | ❌ No | ✅ Yes — production workers shouldn't have test fakes |

### Performance

| Resolution | Time | Notes |
|-----------|------|-------|
| `new MyGuard($queue)` (old) | ~1μs | Direct instantiation |
| `App::make(MyGuard::class, [...])` first call | ~5-20μs | Reflection — cached after first |
| `App::make(MyGuard::class, [...])` subsequent | ~3-8μs | Cached reflection |
| Instance binding (faked) | ~1-2μs | Direct array lookup |

Typical transition = 4 behaviors × ~8μs = ~32μs overhead. DB persist = ~1-5ms. **Overhead < 3%.**

## 4. Constructor DI for Behaviors

Behaviors can now inject service dependencies via constructor — the biggest gain from container resolution.

### Two-Layer DI Architecture

```
┌──────────────────────────────────────────────────┐
│  Constructor DI (Service Dependencies)            │
│  Resolved by Laravel container at instantiation   │
│  PaymentGateway, EmailService, Logger, etc.       │
│                                                    │
│  Where: __construct(ServiceA $a, ServiceB $b)     │
│  When: Behavior is instantiated via App::make()   │
├──────────────────────────────────────────────────┤
│  __invoke DI (Runtime State)                      │
│  Resolved by injectInvokableBehaviorParameters    │
│  ContextManager, EventBehavior, State, History    │
│                                                    │
│  Where: __invoke(ContextManager $ctx, State $s)   │
│  When: Behavior is called during a transition     │
└──────────────────────────────────────────────────┘
```

Services are long-lived, shared, mockable at the container level. State is per-transition, ephemeral, specific to the current execution.

### Before: Service Locator Anti-Pattern

```php
class ProcessPaymentAction extends ActionBehavior
{
    public function __invoke(ContextManager $context): void
    {
        // ❌ Service locator — hard to test, hidden dependency
        $gateway = app(PaymentGateway::class);
        $result = $gateway->charge($context->get('amount'));
        $context->set('transaction_id', $result->id);
    }
}
```

### After: Constructor DI

```php
class ProcessPaymentAction extends ActionBehavior
{
    public function __construct(
        private readonly PaymentGateway $gateway,
        ?Collection $eventQueue = null,
    ) {
        parent::__construct($eventQueue);
    }

    public function __invoke(ContextManager $context): void
    {
        // ✅ Injected — explicit, testable
        $result = $this->gateway->charge($context->get('amount'));
        $context->set('transaction_id', $result->id);
    }
}
```

### Testing Constructor DI Behaviors

```php
it('processes payment via gateway', function () {
    $this->mock(PaymentGateway::class)
        ->shouldReceive('charge')
        ->with(100)
        ->andReturn(new PaymentResult(id: 'txn_123'));

    $state = State::forTesting(context: ['amount' => 100]);
    ProcessPaymentAction::runWithState($state);

    expect($state->context->get('transaction_id'))->toBe('txn_123');
});
```

Or at machine level:

```php
it('processes payment in the full machine', function () {
    $this->mock(PaymentGateway::class)
        ->shouldReceive('charge')->andReturn(new PaymentResult(id: 'txn_456'));

    OrderMachine::test(['amount' => 100])
        ->send('PROCESS_PAYMENT')
        ->assertState('paid')
        ->assertContext('transaction_id', 'txn_456');
});
```

## 5. Fakeable Trait

### Full Implementation

```php
trait Fakeable
{
    /** @var array<class-string, MockInterface> Tracks faked classes for assertions and cleanup. */
    private static array $fakes = [];

    // ─── Creation ─────────────────────────────────

    /**
     * Create a Mockery mock and register it in the container.
     *
     * Uses App::bind() with a closure — NOT App::instance() — because
     * instance bindings are silently bypassed when App::make() receives
     * explicit parameters (e.g., ['eventQueue' => $queue]).
     */
    public static function fake(): MockInterface
    {
        $mock = Mockery::mock(static::class);
        static::$fakes[static::class] = $mock;
        App::bind(static::class, fn () => $mock);

        return $mock;
    }

    /**
     * Create a Mockery spy (allows all calls, records them).
     */
    public static function spy(): MockInterface
    {
        if (isset(static::$fakes[static::class])) {
            return static::$fakes[static::class];
        }

        $spy = Mockery::spy(static::class);
        static::$fakes[static::class] = $spy;
        App::bind(static::class, fn () => $spy);

        return $spy;
    }

    // ─── Expectations ─────────────────────────────

    /**
     * Expect __invoke to be called. Implicitly fakes.
     */
    public static function shouldRun(): Mockery\Expectation|Mockery\CompositeExpectation
    {
        return static::fake()->shouldReceive('__invoke');
    }

    /**
     * Expect __invoke to NOT be called. Implicitly fakes.
     */
    public static function shouldNotRun(): Mockery\Expectation|Mockery\CompositeExpectation
    {
        return static::fake()->shouldNotReceive('__invoke');
    }

    /**
     * Set return value for __invoke. Implicitly fakes.
     */
    public static function shouldReturn(mixed $value): void
    {
        static::fake()->shouldReceive('__invoke')->andReturn($value);
    }

    /**
     * Spy mode — records calls silently, allows all.
     */
    public static function allowToRun(): Mockery\Expectation
    {
        return static::spy()->allows('__invoke');
    }

    // ─── Assertions ───────────────────────────────

    /**
     * Assert __invoke was called at least once.
     */
    public static function assertRan(): void
    {
        if (!isset(static::$fakes[static::class])) {
            throw new RuntimeException('Behavior '.static::class.' was not faked.');
        }

        static::$fakes[static::class]->shouldHaveReceived('__invoke');
    }

    /**
     * Assert __invoke was NOT called.
     */
    public static function assertNotRan(): void
    {
        if (!isset(static::$fakes[static::class])) {
            throw new RuntimeException('Behavior '.static::class.' was not faked.');
        }

        static::$fakes[static::class]->shouldNotHaveReceived('__invoke');
    }

    /**
     * Assert __invoke was called with arguments matching the callback.
     */
    public static function assertRanWith(callable $callback): void
    {
        if (!isset(static::$fakes[static::class])) {
            throw new RuntimeException('Behavior '.static::class.' was not faked.');
        }

        static::$fakes[static::class]
            ->shouldHaveReceived('__invoke')
            ->withArgs($callback);
    }

    /**
     * Assert __invoke was called exactly N times.
     */
    public static function assertRanTimes(int $count): void
    {
        if (!isset(static::$fakes[static::class])) {
            throw new RuntimeException('Behavior '.static::class.' was not faked.');
        }

        static::$fakes[static::class]
            ->shouldHaveReceived('__invoke')
            ->times($count);
    }

    // ─── Inspection ───────────────────────────────

    public static function isFaked(): bool
    {
        return isset(static::$fakes[static::class]);
    }

    public static function getFake(): ?MockInterface
    {
        return static::$fakes[static::class] ?? null;
    }

    // ─── Cleanup ──────────────────────────────────

    /**
     * Reset fake for this specific behavior.
     *
     * Uses offsetUnset() to remove App::bind() bindings.
     * App::forgetInstance() only removes App::instance() bindings.
     */
    public static function resetFakes(): void
    {
        if (isset(static::$fakes[static::class])) {
            $mock = static::$fakes[static::class];

            app()->offsetUnset(static::class);

            foreach (array_keys($mock->mockery_getExpectations()) as $method) {
                $mock->mockery_setExpectationsFor(
                    $method,
                    new Mockery\ExpectationDirector($method, $mock),
                );
            }
            $mock->mockery_teardown();

            unset(static::$fakes[static::class]);
        }
    }

    /**
     * Reset ALL fakes across all behavior classes.
     */
    public static function resetAllFakes(): void
    {
        foreach (static::$fakes as $class => $mock) {
            app()->offsetUnset($class);

            foreach (array_keys($mock->mockery_getExpectations()) as $method) {
                $mock->mockery_setExpectationsFor(
                    $method,
                    new Mockery\ExpectationDirector($method, $mock),
                );
            }
            $mock->mockery_teardown();
        }

        static::$fakes = [];
    }
}
```

### Simplified `InvokableBehavior::run()`

```php
public static function run(mixed ...$args): mixed
{
    return App::make(static::class)(...$args);
}
```

One line. Container returns mock if faked, real instance if not. No branching.

## 6. `runWithState()` + `State::forTesting()`

### The Problem

`::run()` passes raw arguments. The engine uses reflection-based DI via `injectInvokableBehaviorParameters`. A behavior with:

```php
public function __invoke(ContextManager $ctx, EventBehavior $event, State $state): bool
```

...can't be properly tested via `::run()` without manually constructing and ordering all parameters.

### `State::forTesting()` — Lightweight State Factory

```php
// On State class
public static function forTesting(
    array|ContextManager $context = [],
    ?StateDefinition $currentStateDefinition = null,
    ?EventBehavior $currentEventBehavior = null,
): self {
    $contextManager = $context instanceof ContextManager
        ? $context
        : new ContextManager($context);

    return new self(
        context: $contextManager,
        currentStateDefinition: $currentStateDefinition,
        currentEventBehavior: $currentEventBehavior,
        history: new EventCollection(),
    );
}
```

### `InvokableBehavior::runWithState()`

```php
public static function runWithState(
    State $state,
    ?EventBehavior $eventBehavior = null,
    ?array $arguments = null,
): mixed {
    $instance = App::make(static::class);

    $params = static::injectInvokableBehaviorParameters(
        actionBehavior: $instance,
        state: $state,
        eventBehavior: $eventBehavior,
        actionArguments: $arguments,
    );

    return $instance(...$params);
}
```

This method:
- Resolves through container (constructor DI + fake support)
- Uses the exact same `injectInvokableBehaviorParameters` as the engine
- Guarantees: what passes `runWithState()` receives identical parameters in real execution

### Usage Examples

```php
// ─── Simple guard ───
it('checks if count is even', function () {
    $state = State::forTesting(['count' => 4]);
    expect(IsEvenGuard::runWithState($state))->toBeTrue();

    $state = State::forTesting(['count' => 3]);
    expect(IsEvenGuard::runWithState($state))->toBeFalse();
});

// ─── Guard that uses event data ───
it('validates payment currency', function () {
    $state = State::forTesting(['amount' => 100]);
    $event = PaymentEvent::factory(['payload' => ['currency' => 'TRY']]);

    expect(IsCurrencyValidGuard::runWithState($state, $event))->toBeTrue();
});

// ─── Action with constructor DI ───
it('calls payment gateway', function () {
    $this->mock(PaymentGateway::class)
        ->shouldReceive('charge')->with(100)->once()
        ->andReturn(new PaymentResult(id: 'txn_1'));

    $state = State::forTesting(['amount' => 100]);
    ProcessPaymentAction::runWithState($state);

    expect($state->context->get('transaction_id'))->toBe('txn_1');
});

// ─── Calculator with arguments ───
it('multiplies by factor', function () {
    $state = State::forTesting(['value' => 10]);
    MultiplyCalculator::runWithState($state, arguments: ['3']);

    expect($state->context->get('value'))->toBe(30);
});

// ─── Faked behavior ───
it('runs faked via runWithState', function () {
    IncrementAction::shouldRun()->once();

    $state = State::forTesting(['count' => 0]);
    IncrementAction::runWithState($state);

    IncrementAction::assertRan();
    expect($state->context->get('count'))->toBe(0); // Mock didn't modify
});
```

## 7. `Machine::test()` + `TestMachine`

Inspired by Livewire's `Livewire::test()`. A fluent wrapper that keeps test API separate from production API.

### Entry Point

```php
// On Machine class
public static function test(array $context = []): TestMachine
{
    return TestMachine::create(static::class, $context);
}
```

### `TestMachine` Implementation

```php
namespace Tarfinlabs\EventMachine\Testing;

class TestMachine
{
    private Machine $machine;
    private array $fakedBehaviors = [];

    private function __construct(Machine $machine)
    {
        $this->machine = $machine;
    }

    // ═══════════════════════════════════════════
    //  Construction
    // ═══════════════════════════════════════════

    /** From a Machine subclass. */
    public static function create(string $machineClass, array $context = []): self
    {
        return new self($machineClass::create($context));
    }

    /** From an inline definition (no persistence, no Machine class). */
    public static function define(array $config, array $behavior = []): self
    {
        $definition = MachineDefinition::define(config: $config, behavior: $behavior);
        $machine = Machine::withDefinition($definition);
        $machine->state = $definition->initialStateDefinition->transitionTo($definition);
        return new self($machine);
    }

    /** Wrap an existing machine instance. */
    public static function for(Machine $machine): self
    {
        return new self($machine);
    }

    // ═══════════════════════════════════════════
    //  Configuration
    // ═══════════════════════════════════════════

    /** Fake specific behaviors. Selective faking à la Bus::fake([...]). */
    public function faking(array $behaviors): self
    {
        foreach ($behaviors as $behavior) {
            $behavior::fake();
            $this->fakedBehaviors[] = $behavior;
        }
        return $this;
    }

    /** Disable persistence for this test. */
    public function withoutPersistence(): self
    {
        $this->machine->definition->shouldPersist = false;
        return $this;
    }

    /** Run parallel regions synchronously. */
    public function withoutParallelDispatch(): self
    {
        if (isset($this->machine->definition->config['parallel_dispatch'])) {
            $this->machine->definition->config['parallel_dispatch']['enabled'] = false;
        }
        return $this;
    }

    // ═══════════════════════════════════════════
    //  Actions
    // ═══════════════════════════════════════════

    /** Send an event. Accepts EventBehavior, array, or string shorthand. */
    public function send(EventBehavior|array|string $event): self
    {
        if (is_string($event)) {
            $event = ['type' => $event];
        }
        $this->machine->send($event);
        return $this;
    }

    /** Send multiple events in sequence. */
    public function sendMany(array $events): self
    {
        foreach ($events as $event) {
            $this->send($event);
        }
        return $this;
    }

    // ═══════════════════════════════════════════
    //  State Assertions
    // ═══════════════════════════════════════════

    public function assertState(string|array $expected): self
    {
        expect($this->machine->state->matches($expected))->toBeTrue(
            "Expected state [{$expected}] but got [".implode(', ', $this->machine->state->value)."]"
        );
        return $this;
    }

    public function assertNotState(string $state): self
    {
        expect($this->machine->state->matches($state))->toBeFalse(
            "Expected NOT to be in state [{$state}]"
        );
        return $this;
    }

    public function assertFinished(): self
    {
        expect($this->machine->state->currentStateDefinition?->type)
            ->toBe(StateDefinitionType::Final, 'Expected a final state');
        return $this;
    }

    // ═══════════════════════════════════════════
    //  Context Assertions
    // ═══════════════════════════════════════════

    public function assertContext(string $key, mixed $expected): self
    {
        expect($this->machine->state->context->get($key))->toBe($expected,
            "context[{$key}]: expected ".json_encode($expected)
        );
        return $this;
    }

    public function assertContextHas(string $key): self
    {
        expect($this->machine->state->context->has($key))->toBeTrue(
            "Expected context to have [{$key}]"
        );
        return $this;
    }

    public function assertContextMissing(string $key): self
    {
        expect($this->machine->state->context->has($key))->toBeFalse(
            "Expected context NOT to have [{$key}]"
        );
        return $this;
    }

    public function assertContextIncludes(array $subset): self
    {
        foreach ($subset as $key => $value) {
            $this->assertContext($key, $value);
        }
        return $this;
    }

    // ═══════════════════════════════════════════
    //  Transition Assertions
    // ═══════════════════════════════════════════

    /** Send + assertState in one call. */
    public function assertTransition(array|string $event, string $expectedState): self
    {
        return $this->send($event)->assertState($expectedState);
    }

    /** Assert an event is guarded (state unchanged). */
    public function assertGuarded(array|string $event): self
    {
        $before = $this->machine->state->value;
        $this->send($event);
        expect($this->machine->state->value)->toBe($before,
            'Expected event to be guarded, but a transition occurred'
        );
        return $this;
    }

    /** Assert an event raises MachineValidationException. */
    public function assertValidationFailed(array|string $event, ?string $errorKey = null): self
    {
        try {
            $this->send($event);
            expect(false)->toBeTrue('Expected MachineValidationException');
        } catch (MachineValidationException $e) {
            if ($errorKey !== null) {
                expect($e->errors())->toHaveKey($errorKey);
            }
        }
        return $this;
    }

    // ═══════════════════════════════════════════
    //  History Assertions
    // ═══════════════════════════════════════════

    public function assertHistoryContains(string ...$eventTypes): self
    {
        $history = $this->machine->state->history->pluck('type')->toArray();
        foreach ($eventTypes as $type) {
            expect($history)->toContain($type);
        }
        return $this;
    }

    public function assertGuardPassed(string $guardClass): self
    {
        $name = class_basename($guardClass);
        $found = $this->machine->state->history
            ->pluck('type')
            ->contains(fn ($t) => str_contains($t, "guard.{$name}.pass"));
        expect($found)->toBeTrue("Expected guard [{$guardClass}] to pass");
        return $this;
    }

    public function assertGuardFailed(string $guardClass): self
    {
        $name = class_basename($guardClass);
        $found = $this->machine->state->history
            ->pluck('type')
            ->contains(fn ($t) => str_contains($t, "guard.{$name}.fail"));
        expect($found)->toBeTrue("Expected guard [{$guardClass}] to fail");
        return $this;
    }

    // ═══════════════════════════════════════════
    //  Path Assertions
    // ═══════════════════════════════════════════

    /**
     * @param  array<array{event: string|array, state: string, context?: array}>  $steps
     */
    public function assertPath(array $steps): self
    {
        foreach ($steps as $i => $step) {
            $this->send($step['event']);
            expect($this->machine->state->matches($step['state']))->toBeTrue(
                "Step {$i}: expected [{$step['state']}], got [".implode(', ', $this->machine->state->value)."]"
            );
            foreach ($step['context'] ?? [] as $key => $value) {
                expect($this->machine->state->context->get($key))->toBe($value,
                    "Step {$i}: context[{$key}]"
                );
            }
        }
        return $this;
    }

    // ═══════════════════════════════════════════
    //  Parallel Assertions
    // ═══════════════════════════════════════════

    public function assertRegionState(string $regionId, string $expectedState): self
    {
        $match = collect($this->machine->state->value)
            ->first(fn ($v) => str_contains($v, $regionId));
        expect($match)->not->toBeNull("No active state in region [{$regionId}]")
            ->and($match)->toContain($expectedState);
        return $this;
    }

    public function assertAllRegionsCompleted(): self
    {
        $result = $this->machine->definition->areAllRegionsFinal(
            $this->machine->state,
            $this->machine->state->currentStateDefinition,
        );
        expect($result)->toBeTrue('Expected all parallel regions to be final');
        return $this;
    }

    public function assertParallelDispatched(?int $count = null): self
    {
        expect($this->machine->dispatched)->toBeTrue('Expected parallel dispatch');
        if ($count !== null) {
            Queue::assertPushed(ParallelRegionJob::class, $count);
        }
        return $this;
    }

    public function assertRegionDispatched(string $regionId): self
    {
        Queue::assertPushed(
            ParallelRegionJob::class,
            fn ($job) => str_contains($job->regionStateId, $regionId),
        );
        return $this;
    }

    // ═══════════════════════════════════════════
    //  Behavior Assertions (convenience)
    // ═══════════════════════════════════════════

    public function assertBehaviorRan(string $class): self
    {
        $class::assertRan();
        return $this;
    }

    public function assertBehaviorNotRan(string $class): self
    {
        $class::assertNotRan();
        return $this;
    }

    // ═══════════════════════════════════════════
    //  Accessors
    // ═══════════════════════════════════════════

    public function machine(): Machine { return $this->machine; }
    public function state(): State { return $this->machine->state; }

    // ═══════════════════════════════════════════
    //  Cleanup
    // ═══════════════════════════════════════════

    public function resetFakes(): self
    {
        foreach ($this->fakedBehaviors as $behavior) {
            $behavior::resetFakes();
        }
        $this->fakedBehaviors = [];
        return $this;
    }
}
```

## 8. `EventBehavior::factory()`

Events are **data**, not **behavior** — you construct them easily, not mock them. Factory follows `Model::factory()`.

### On the Base Class

```php
abstract class EventBehavior extends Data
{
    // ...existing code...

    /**
     * Create an event instance for testing with sensible defaults.
     * Override in concrete classes for domain-specific defaults.
     */
    public static function factory(array $attributes = []): static
    {
        return static::from(array_merge([
            'type'    => static::getType(),
            'payload' => [],
            'version' => 1,
        ], $attributes));
    }
}
```

### Concrete Override

```php
class PaymentEvent extends EventBehavior
{
    public static function getType(): string { return 'PROCESS_PAYMENT'; }

    public static function factory(array $attributes = []): static
    {
        return parent::factory(array_merge([
            'payload' => ['amount' => 100, 'currency' => 'TRY'],
        ], $attributes));
    }
}
```

### Usage

```php
$event = PaymentEvent::factory();
$event = PaymentEvent::factory(['payload' => ['amount' => 0]]);

// With runWithState
$state = State::forTesting(['balance' => 500]);
expect(HasSufficientBalanceGuard::runWithState($state, PaymentEvent::factory()))->toBeTrue();

// With Machine::test
OrderMachine::test()->send(PaymentEvent::factory()->toArray());
```

## 12. Design Principles

### 1. Container-First

Every behavior is a container citizen. Constructor DI, automatic resolution, mock binding — all standard Laravel patterns.

### 2. Two-Layer DI

| Layer | What | Where | Resolved By |
|-------|------|-------|-------------|
| Services | `PaymentGateway`, `Logger` | `__construct()` | Laravel Container |
| State | `ContextManager`, `EventBehavior`, `State` | `__invoke()` | `injectInvokableBehaviorParameters` |

### 3. Real by Default, Opt-in Faking

No `Machine::fake()` that disables everything. Each behavior is individually, explicitly faked. Unfaked behaviors run with real logic.

### 4. Test Wrapper, Not Production Pollution

`TestMachine` is a separate class. `Machine::test()` is the only bridge. Assertions never leak into production `Machine`.

### 5. Factory for Data, Mock for Behavior

| Construct | Pattern | Why |
|-----------|---------|-----|
| `InvokableBehavior` (Guard, Action, Calculator) | `::fake()` / `::shouldReturn()` | Has behavior to mock |
| `EventBehavior` | `::factory()` | Data object — construct, don't mock |
| `State` | `::forTesting()` | Lightweight construction for tests |

### 6. Layered Testing Pyramid

```
        ╱╲
       ╱  ╲    Path: assertPath(), full lifecycles
      ╱────╲
     ╱      ╲   Transition: assertTransition(), assertGuarded()
    ╱────────╲
   ╱          ╲  Behavior: runWithState(), shouldReturn(), assertRan()
  ╱────────────╲
```

### 7. Minimal Production Overhead

Container resolution adds ~3-8μs per behavior (after first-call reflection cache). For a typical transition with 4 behaviors, this is ~32μs — under 3% of DB persist time. In exchange: full constructor DI, automatic fake resolution, Laravel-idiomatic architecture.

# Fakeable Behaviors

## How It Works

Fakes are registered via `App::bind($class, fn() => $mock)` — NOT `App::instance()`.

> **Why not `App::instance()`?** Instance bindings are silently bypassed when
> `App::make()` receives explicit parameters. Since the engine always calls
> `App::make($class, ['eventQueue' => $queue])`, fakes registered via
> `App::instance()` would never be returned.

## Creating Fakes

<!-- doctest-attr: ignore -->
​```php
// Mock — strict, expectations must be set
$mock = ProcessOrderAction::fake();
$mock->shouldReceive('__invoke')->once();

// Spy — permissive, records all calls silently
$spy = ProcessOrderAction::spy();
// ...run machine...
$spy->shouldHaveReceived('__invoke');
​```

## Setting Expectations

<!-- doctest-attr: ignore -->
​```php
// Expect __invoke to be called (implicitly fakes)
ProcessOrderAction::shouldRun()->once();

// Expect __invoke to NOT be called (implicitly fakes)
SendEmailAction::shouldNotRun();

// Preset return value (for guards)
IsValidGuard::shouldReturn(true);
IsValidGuard::shouldReturn(false);

// Spy mode — allow all calls, record them
ProcessOrderAction::allowToRun();
​```

## Assertions

<!-- doctest-attr: ignore -->
​```php
ProcessOrderAction::assertRan();
ProcessOrderAction::assertNotRan();
ProcessOrderAction::assertRanTimes(3);
ProcessOrderAction::assertRanWith(fn($ctx) => $ctx->get('amount') === 100);
​```

## Selective Faking — Bus::fake() Style

<!-- doctest-attr: ignore -->
​```php

    ->faking([SendEmailAction::class])
    ->send('SUBMIT')

## Before/After Comparison

### Before — Service Locator (anti-pattern)
​```php
class ProcessPaymentAction extends ActionBehavior {
    public function __invoke(ContextManager $context): void {
        $gateway = app(PaymentGateway::class);  // hidden dependency
        $result = $gateway->charge($context->get('amount'));
    }
}
​```

### After — Constructor DI
​```php
class ProcessPaymentAction extends ActionBehavior {
    public function __construct(
        private readonly PaymentGateway $gateway,
        ?Collection $eventQueue = null,
    ) {
        parent::__construct($eventQueue);
    }

    public function __invoke(ContextManager $context): void {
        $result = $this->gateway->charge($context->get('amount'));  // explicit
    }
}
​```

| Approach | When | Example |
|----------|------|---------|
| Mock the **service** | Test behavior logic with controlled service responses | `$this->mock(PaymentGateway::class)` |
| Mock the **behavior** | Test machine flow, skip behavior internals | `ProcessPaymentAction::fake()` |
| Mock **neither** | E2E with real services (or test doubles in ServiceProvider) | Full integration test |

### Writing Guidelines

All testing docs must follow:

1. **Code blocks**: Use `<!-- doctest-attr: ignore -->` for test examples (Pest framework dependency)
2. **Pattern**: Open with "why", show simplest complete example first, then expand to advanced
3. **Cross-links**: Bidirectional — hub pages link to relevant behavior docs, behavior docs link to hub (see matrix above)
4. **Three levels**: Each behavior spoke shows isolated → faked → machine-level with concrete code
5. **Real examples**: Use actual test stubs from `tests/Stubs/` wherever possible — don't invent classes that don't exist
6. **VitePress tips**: Use `::: tip` callouts for cross-references and important notes
7. **Consistency**: Every code example must be complete enough to understand without surrounding context
