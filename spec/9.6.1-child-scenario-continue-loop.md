# Fix: executeChildScenario Missing @continue Loop

**Status:** Draft
**Date:** 2026-04-02

## Problem

`ScenarioPlayer::executeChildScenario()` only calls `$childMachine->start()`, which runs the `@always` chain and outcome interception but stops at the first interactive state. It has **no `@continue` loop** — unlike `execute()` and `executeContinuation()`.

Outcome interception via `handleScenarioOutcome()` **works correctly** during `start()` — job actor states with outcomes advance through the chain. The chain breaks only when reaching an **interactive state with `@continue`** — the machine pauses and `start()` returns with no loop to process `@continue` directives.

### The Findeks Example

```
AtAwaitingPinFromStartScenario (child, FindeksMachine):
  plan:
    'checking_existing_report' => [HasExistingReportGuard => false]        ← @always chain
    'querying_phones'          => ['outcome' => '@done', ...]              ← outcome intercepted ✓
    'awaiting_report_request'  => ['@continue' => ReportRequestedEvent]    ← STUCK HERE (no loop)
    'requesting'               => ['outcome' => '@done', ...]              ← never reached
    'polling'                  => ['outcome' => '@done', IsPinRequired => true]
```

Flow during `executeChildScenario()`:
```
idle → @always → checking_existing_report → @always (guard=false)
→ querying_phones → outcome @done (intercepted ✓) → awaiting_report_request
→ interactive state, start() returns — NO @continue loop → returns null
```

**Confirmed root cause:** `executeChildScenario()` is missing the `@continue` loop. TDD test written and FAILING at `tests/Features/ScenarioPlayerChildScenarioTest.php` — "child scenario with job actor outcomes".

## Proposed Solution

Add a `@continue` loop to `executeChildScenario()` after `$childMachine->start()`. Three issues must be resolved:

1. **Outcome restore timing** — parent outcomes must be restored AFTER the loop, not before
2. **classifyPlanValues() side effects** — must not overwrite parent's `self::$outcomes`
3. **Helper methods are instance, not static** — promote to static (no `$this` usage)

## Implementation

### 1. Promote helper methods to static

`resolveCurrentRoute()`, `findContinueForState()`, `parseContinueValue()`, `resolveEventType()` reference no instance state (`$this`). Promote from `private function` to `private static function`. Zero-risk refactor — all callers already accessible.

### 2. Replace manual parsing with non-side-effecting classification

Current code (lines 367-375) manually parses outcomes and child scenarios but skips `@continue` overrides. Replace with classification logic that returns all three categories WITHOUT writing to `self::$outcomes`/`self::$childScenarios`:

```php
// Extract child plan classification (outcomes, childScenarios, overrides with @continue)
// Do NOT call classifyPlanValues() — it writes to self::$outcomes as side effect.
// Instead, inline the classification to local variables only.
$childOutcomes       = [];
$childChildScenarios = [];
$childOverrides      = [];

foreach ($childPlan as $stateRoute => $value) {
    if (is_string($value) && str_starts_with($value, '@')) {
        $childOutcomes[$stateRoute] = $value;
    } elseif (is_string($value) && class_exists($value) && is_subclass_of($value, MachineScenario::class)) {
        $childChildScenarios[$stateRoute] = $value;
    } elseif (is_array($value) && isset($value['outcome'])) {
        $childOutcomes[$stateRoute] = $value;
        // Also extract behavior overrides from outcome array
        $behaviorKeys = array_filter(
            $value,
            fn (mixed $v, int|string $k): bool => is_string($k) && $k !== 'outcome' && $k !== 'output' && class_exists($k),
            ARRAY_FILTER_USE_BOTH,
        );
        if ($behaviorKeys !== []) {
            $childOverrides[$stateRoute] = $behaviorKeys;
        }
    } elseif (is_array($value)) {
        $childOverrides[$stateRoute] = $value;
    }
}
```

### 3. Move parent restore AFTER @continue loop

Current order: `start()` → restore parent outcomes → return.
New order: `start()` → @continue loop (child outcomes active) → restore parent outcomes → return.

```php
// Merge child outcomes into static storage (child takes precedence)
self::$outcomes       = array_merge(self::$outcomes, $childOutcomes);
self::$childScenarios = array_merge(self::$childScenarios, $childChildScenarios);

$childMachine->start();
$childState = $childMachine->state;

// @continue loop — advance through interactive states (child outcomes still active)
$continueCount = 0;
$maxDepth      = (int) config('machine.max_transition_depth', 100);

while ($continueCount < $maxDepth) {
    $currentRoute = self::resolveCurrentRoute($childState);
    $continue     = self::findContinueForState($currentRoute, $childOverrides);

    if ($continue === null) {
        break;
    }

    $continueCount++;
    [$eventClass, $payload] = self::parseContinueValue($continue);

    try {
        $childState = $childMachine->send([
            'type'    => self::resolveEventType($eventClass),
            'payload' => $payload,
        ]);
    } catch (\Throwable $e) {
        // Restore parent outcomes before re-throwing
        self::$outcomes       = $parentOutcomes;
        self::$childScenarios = $parentChildScenarios;

        throw ScenarioFailedException::continueEventFailed(
            state: $currentRoute,
            event: $eventClass,
            reason: $e->getMessage(),
        );
    }
}

// Restore parent outcomes AFTER loop completes
self::$outcomes       = $parentOutcomes;
self::$childScenarios = $parentChildScenarios;
```

### 4. Error handling

`@continue` failures in child scenarios throw `ScenarioFailedException::continueEventFailed()` — same contract as `execute()`. Parent outcomes are restored in the catch block before re-throwing.

## What Gets Added

| Component | Change |
|-----------|--------|
| `ScenarioPlayer::executeChildScenario()` | @continue loop, parent outcome restore after loop, @continue error handling |
| `ScenarioPlayer::resolveCurrentRoute()` | `private` → `private static` |
| `ScenarioPlayer::findContinueForState()` | `private` → `private static` |
| `ScenarioPlayer::parseContinueValue()` | `private` → `private static` |
| `ScenarioPlayer::resolveEventType()` | `private` → `private static` |

## What Doesn't Change

- `execute()` — unchanged
- `executeContinuation()` — unchanged (already calls instance methods, will call static methods after promotion)
- `handleScenarioOutcome()` — unchanged (outcome interception already works during start())
- `handleMachineInvoke()` — unchanged
- Child scenarios with only @always chains (no @continue) — loop exits immediately, behavior identical
