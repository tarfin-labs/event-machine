# Raise Actor Auto-Propagation — v8.0.0

## Problem

When an action calls `raise()`, the actor must be manually passed every time:

```php
$this->raise(new ApprovedEvent(
    payload: $request->toArray(),
    actor: $event->actor($context),  // ← boilerplate, easy to forget
));
```

Forgetting `actor:` in one `raise()` silently drops the actor for the rest of the chain (19 instances across 8 backend action files).

## Prerequisite

PR #127 (`State::$triggeringEvent`) must be merged first — it gives us a reliable actor source for propagation.

## Solution

Auto-propagate actor in `MachineDefinition::initializeEvent()`. When a raised event has no actor and `$state->triggeringEvent` has one, inherit it.

### Flow

```
send(['type' => 'SUBMIT', 'actor' => $client])
  ↓
transition() resets triggeringEvent to null (new macrostep)
  ↓
initializeEvent(SUBMIT) — triggeringEvent null → no propagation
  ↓
PR #127 sets triggeringEvent = SUBMIT(actor: $client)
  ↓
Action calls raise(new ApprovedEvent(payload: [...]))
  ↓
Event queue: initializeEvent(ApprovedEvent)
  — triggeringEvent has actor → setActor($client)
  ↓
Recursive transition(ApprovedEvent)
  — resets triggeringEvent, PR #127 sets it to ApprovedEvent(actor: $client)
  ↓
Next raise inherits from ApprovedEvent → chain preserved
```

## Implementation

### 1. Add `setActor()` to EventBehavior

```php
// src/Behavior/EventBehavior.php

/** @internal Framework use only — called by initializeEvent() for actor propagation. */
public function setActor(mixed $actor): void
{
    $this->actor = $actor;
}
```

### 2. Reset triggeringEvent for new macrosteps

Inside `transition()`, the order is:

```
line 2673: initializeEvent()              ← propagation reads triggeringEvent
line 2676: triggeringEvent = eventBehavior ← PR #127 sets it AFTER
```

`initializeEvent()` sees the triggeringEvent from the **previous** transition. For raised events this is correct (the previous event triggered the raise). But for consecutive `send()` calls, the previous macrostep's actor would leak:

```php
$machine->send(['type' => 'FOO', 'actor' => $userA]);
$machine->send(['type' => 'BAR']);  // BAR would get $userA's actor — wrong!
```

Fix: reset triggeringEvent at the start of `transition()`, before `getScenarioStateIfAvailable()` (which also calls `initializeEvent()` internally):

```php
// src/Definition/MachineDefinition.php — inside transition()
if ($state instanceof State) {
    // Reset triggeringEvent for new macrostep (not for @always — they inherit by design)
    $eventType = is_array($event) ? ($event['type'] ?? null) : $event->type;
    if ($eventType !== TransitionProperty::Always->value) {
        $state->triggeringEvent = null;
    }

    $state = $this->getScenarioStateIfAvailable(state: $state, eventBehavior: $event);
} else {
    $state = $this->getInitialState(event: $event);
}
```

### 3. Propagate actor in `initializeEvent()`

Unify the three early-return paths, add propagation at the end:

```php
// src/Definition/MachineDefinition.php
protected function initializeEvent(
    EventBehavior|array $event,
    State $state
): EventBehavior {
    if ($event instanceof EventBehavior) {
        $eventBehavior = $this->resolveEventBehaviorThroughRegistry($event, $state);
    } elseif (isset($state->currentStateDefinition->machine->behavior[BehaviorType::Event->value][$event['type']])) {
        $eventDefinitionClass = $state
            ->currentStateDefinition
            ->machine
            ->behavior[BehaviorType::Event->value][$event['type']];

        $eventBehavior = $eventDefinitionClass::validateAndCreate($event);
    } else {
        $eventBehavior = EventDefinition::from($event);
    }

    // Auto-propagate actor from triggering event if not explicitly set
    $triggeringActor = $state->triggeringEvent?->actor($state->context);

    if ($eventBehavior->actor($state->context) === null && $triggeringActor !== null) {
        $eventBehavior->setActor($triggeringActor);
    }

    return $eventBehavior;
}
```

Note: `createEventBehavior()` is a public wrapper around `initializeEvent()`. After this change it also propagates actor. ParallelRegionJob uses it explicitly (step 4) to initialize raised events before `transition()` resets triggeringEvent.

### 4. Carry actor through ParallelRegionJob

Parallel region entry actions are a continuation of the same macrostep — the user's event triggered the parallel state entry. Actor should flow.

**Why only ParallelRegionJob?** All other async jobs are independent macrosteps:

| Job | Why no propagation needed |
|-----|--------------------------|
| SendToMachineJob | Event carries its own actor |
| ChildMachineCompletionJob | @done/@fail is a lifecycle event, not user-triggered |
| ListenerJob | Async side-effect, separate macrostep |
| Timeout jobs | System-triggered, no user actor |
| ChildMachineJob | Fresh child, independent actor context |

**Dispatch site** — pass actor in job constructor:

```php
// src/Actor/Machine.php — dispatchPendingParallelJobs()
$job = new ParallelRegionJob(
    machineClass: $this->definition->machineClass,
    rootEventId: $rootEventId,
    regionId: $dispatch['region_id'],
    initialStateId: $dispatch['initial_state_id'],
    contextAtDispatch: $this->state->context->data,
    triggeringActor: $this->state->triggeringEvent?->actor($this->state->context),
);
```

`SerializesModels` on the job handles Model serialization (backend actor is typically `Client`).

**Job handle** — restore actor on freshMachine, initialize events before transition:

```php
// src/Jobs/ParallelRegionJob.php — inside handle(), before raised events loop

// Restore actor for raised event propagation
if ($this->triggeringActor !== null) {
    $carrier = new EventDefinition(type: InternalEvent::PARALLEL_REGION_ENTER->value);
    $carrier->setActor($this->triggeringActor);
    $freshMachine->state->triggeringEvent = $carrier;
}

// 13. Process raised events
foreach ($raisedEvents as $event) {
    // Initialize before transition — mirrors framework's internal event queue pattern
    $eventBehavior = $freshMachine->definition->createEventBehavior($event, $freshMachine->state);
    $freshMachine->state = $freshMachine->definition->transition($eventBehavior, $freshMachine->state);
}
```

**Why createEventBehavior() before transition()?** Our reset in `transition()` nulls triggeringEvent before `initializeEvent()` runs. By calling `createEventBehavior()` (which calls `initializeEvent()`) BEFORE `transition()`, the actor is propagated while triggeringEvent is still set. This is the same pattern the framework uses for internal event queue processing.

### Propagation rules

| Scenario | Result |
|----------|--------|
| External event via send() | No propagation — triggeringEvent reset to null |
| raise() without actor | **Inherited** from triggeringEvent |
| raise() with explicit actor | Explicit wins — `actor($context) !== null` skips injection |
| Chained raise (A → B → C) | Each inherits from previous — triggeringEvent updated at each recursive transition() |
| Timer / @done / @fail → raise | No propagation — these events have no actor, so triggeringEvent actor is null |
| Event class with `actor()` override | Override wins — returns non-null, skips injection |
| Consecutive send() calls | No leaking — triggeringEvent reset at start of each transition() |
| Parallel dispatch (async regions) | **Propagated** — actor carried via job payload, restored on freshMachine |

## Test Scenarios

### Core

1. **Inherits actor**: send(actor: A) → raise without actor → raised event has actor A
2. **Explicit actor wins**: send(actor: A) → raise(actor: B) → raised event has actor B
3. **No actor anywhere**: send without actor → raise without actor → both null
4. **Chain preserves**: send(actor: A) → raise X → X's action raises Y → Y has actor A
5. **Chain explicit override**: send(actor: A) → raise(actor: B) → B's action raises C → C has actor B

### Edge Cases

6. **Consecutive send() no leak**: send(actor: A) → second send(BAR without actor) → BAR has no actor
7. **Timer → raise**: timer fires → action raises → raised event actor is null
8. **@done → raise**: child completes → parent action raises → raised event actor is null
9. **Event with actor() override**: raise event with override → override value preserved
10. **Initial @always → raise**: machine created, initial @always, action raises → triggeringEvent null → no propagation
11. **raise() into @always**: raise FOO → target has @always → @always behaviors see FOO's actor (PR #127)
12. **Parallel dispatch preserves actor**: send(actor: A) → enters parallel state → region entry action raises → raised event has actor A (via job payload)
