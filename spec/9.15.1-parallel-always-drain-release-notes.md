# Release Notes — 9.15.1 startingAt @always Drain in Parallel Regions

Bug-fix release for the 9.15.0 testing DX feature. No consumer changes required; upgrade with `composer update tarfin-labs/event-machine`.

## Fixed

- **`Machine::startingAt()` now drains `@always` transitions on parallel region leaves.** The 9.15.0 drain only inspected `currentStateDefinition`, which for targets inside parallel regions is anchored on the parallel ancestor — a region leaf's `@always` was never evaluated and the machine parked at a configuration the real machine can never rest in. The drain now scans every active leaf in the state's value array, mirroring the parallel-entry drain the runtime already performs: the region whose leaf `@always` passes routes (running the transition's actions), while sibling regions stay at their leaves.

  ```php
  // 9.15.0: parked at calculating (region @always never seen)
  // 9.15.1: retailer region routes, documents region stays put
  CarSalesMachine::startingAt('processing.retailer.calculating')
      ->assertRegionState('retailer', 'awaiting_options')
      ->assertRegionState('documents', 'awaiting_docs');
  ```

  The `guards:` park escape hatch works in parallel regions too — pin the region leaf's `@always` guards false to park at the transient leaf.
