# Preserve Original Event Through @always Transitions ‚Äî v8.0.0

## Problem

When a state machine has chained `@always` transitions, the original triggering event's payload is lost. Guards, actions, and calculators on `@always` transitions receive a synthesized event with `type: '@always'` and no payload.

```
idle --[UserSubmittedEvent {tckn, phone}]--> routing --[@always]--> eligibility --[@always]--> verification
                                                        ‚Üë
                                            $event->type === '@always'
                                            $event->payload === null  ‚Üê lost!
```

This forces every piece of event data to be explicitly copied into context before any @always chain.

## Spec & Framework Consensus

| Framework | Preserves original event? | Notes |
|-----------|:---:|-------|
| SCXML W3C | Yes | `_event` stays bound to last matched event |
| XState v5 | **Yes** | PR #2824 ‚Äî `NullEvent` removed entirely, all behaviors (entry actions, guards, etc.) receive the original event |
| Spring SM | Yes | `StateContext` with original `Message` passed through |
| Harel | N/A | Not specified in original formalism |
| Python transitions | No | Different model ‚Äî no eventless transitions |
| **EventMachine (current)** | **No** | Matches XState v4's broken behavior |

**Consensus: the original event should be preserved.** EventMachine's current behavior is a known limitation that matches XState v4's acknowledged bug (#890).

## Solution: Transparent Event Preservation

### Why the synthetic @always event must stay

The routing mechanism (`findTransitionDefinition()`) uses string-based key lookup: `$transitionDefinitions[$eventBehavior->type]`. The `@always` transition is keyed by the string `'@always'` in the definitions array. Eliminating the synthetic event would require restructuring the entire routing system ‚Äî not worth it. Instead, we preserve the synthetic event for routing but inject the original event into behaviors.

### Approach

1. Track the triggering event on `State` as a transient property
2. Routing continues to use synthetic `@always` event for `findTransitionDefinition()`
3. Behaviors (actions, guards, calculators) receive the **triggering event** via parameter injection
4. `$state->currentEventBehavior` is NOT changed ‚Äî the internal event logging system (action start/end markers) overwrites it during behavior execution, making it unreliable for this purpose. Parameter injection is the sole access path.
5. Internal event history still records `@always` ‚Äî observability unchanged

### Implementation

#### 1. Add `triggeringEvent` to State

```php
// src/Actor/State.php
public ?EventBehavior $triggeringEvent = null;
```

Transient runtime property ‚Äî same pattern as `$lastChildDoneRoute` and `$forwardedChildState`:
- Not in `jsonSerialize()` return array ‚Üí not serialized
- Not in queue job payloads ‚Üí State is always reconstructed from DB via `rootEventId`
- Survives the entire synchronous @always chain within a single macrostep

#### 2. Set triggering event in `transition()`

In `MachineDefinition::transition()`, after `initializeEvent()`:

```php
$eventBehavior = $this->initializeEvent($event, $state);

// Track the triggering event for @always chains
if ($eventBehavior->type !== TransitionProperty::Always->value) {
    $state->triggeringEvent = $eventBehavior;
}
```

#### 3. Inject triggering event into behaviors

In `InvokableBehavior::injectInvokableBehaviorParameters()`, when resolving the `EventBehavior` parameter:

```php
$effectiveEvent = $eventBehavior;
if ($eventBehavior !== null
    && $eventBehavior->type === TransitionProperty::Always->value
    && $state->triggeringEvent !== null) {
    $effectiveEvent = $state->triggeringEvent;
}
```

#### 4. `$state->currentEventBehavior` ‚Äî NOT changed

`$state->currentEventBehavior` is NOT updated during @always. The internal event logging system (`setInternalEventBehavior` with `ACTION_START`, `ACTION_END` markers) overwrites `currentEventBehavior` during behavior execution. This means `$state->currentEventBehavior` inside an action is always the internal action marker, not the user event ‚Äî regardless of @always. Parameter injection via `InvokableBehavior::injectInvokableBehaviorParameters()` is the sole reliable access path for the triggering event.

#### 5. Edge cases

**Init with @always (no triggering event):**
`triggeringEvent` stays null. Behaviors receive null event. No change from current behavior.

**raise() ‚Üí @always:**
Raised event enters `transition()` ‚Üí sets `triggeringEvent` ‚Üí @always chain carries it. Correct.

**Timer @after/@every ‚Üí @always:**
Timer event is a real trigger ‚Üí sets `triggeringEvent` ‚Üí @always chain carries it. Correct.

**Nested compound @done ‚Üí @always:**
`processCompoundOnDone` does NOT call `transition()` ‚Äî it handles state changes directly (exit/entry actions, `$state->setValues()`). After it returns, `transition()` checks for @always. At this point `triggeringEvent` is still the macrostep's original event ‚Äî correct.

> **Pre-existing concern (out of scope):** `processCompoundOnDone` updates `$state->value` but may not update `$state->currentStateDefinition`. The @always check uses `$newState->currentStateDefinition->id` ‚Äî could check wrong state if @done changed target. Should be investigated separately.

**Async parallel dispatch:**
`ParallelRegionJob` reconstructs State from DB ‚Üí `triggeringEvent` is null (transient). Entry actions already receive null event in async mode. No regression.

**Reset timing ‚Äî self-maintaining:**
No explicit cleanup needed. Each real event entering `transition()` overwrites `triggeringEvent`. State reconstructed from DB starts with null.

## What Changes

| Aspect | v7 | v8 |
|--------|----|----|
| `$event->type` in @always behavior | `'@always'` | Original event type (e.g., `'USER_SUBMITTED'`) |
| `$event->payload` in @always behavior | `null` | Original event payload |
| `$event->actor()` in @always behavior | Derived from context | Derived from original event |
| `$state->currentEventBehavior` in @always | Synthetic `@always` event | Unchanged ‚Äî internal logging overwrites it |
| Internal event history | `@always` | `@always` (unchanged) |
| `findTransitionDefinition()` routing | Uses `@always` | Uses `@always` (unchanged) |
