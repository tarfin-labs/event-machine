## Issue 1: assertRaised() for Isolated Action Tests

### Problem

Actions that call `$this->raise()` cannot be easily asserted after `runWithState()`. The return value IS the eventQueue Collection, but tests don't use it — they resort to manual reflection or `dispatchTo()` workarounds.

### Solution

Add static assertion methods to `InvokableBehavior` following the existing `assertRan()` / `assertNotRan()` pattern from the Fakeable trait.

`runWithState()` already creates and returns eventQueue. The only change: **save it to a static property** so assertions can read it.

```php
// In runWithState() — add one line:
self::$raisedEventsPerClass[static::class] = $eventQueue;

// New assertion methods:
CheckProtocolAction::runWithState($state);
CheckProtocolAction::assertRaised(ProtocolUndecidedEvent::class);   // FQCN
CheckProtocolAction::assertRaised('PROTOCOL_UNDECIDED');            // event type string
CheckProtocolAction::assertNotRaised(ProtocolRejectedEvent::class);
CheckProtocolAction::assertRaisedCount(1);
CheckProtocolAction::assertNothingRaised();
```

### Implementation

```php
// InvokableBehavior — per-class indexed (same pattern as Fakeable::$fakes):
private static array $raisedEventsPerClass = [];

// In runWithState() — before return:
self::$raisedEventsPerClass[static::class] = $eventQueue;

// New methods:
public static function assertRaised(string $eventTypeOrClass): void
public static function assertNotRaised(string $eventTypeOrClass): void
public static function assertRaisedCount(int $expected): void
public static function assertNothingRaised(): void

// Event matching (private helper):
// Matches by: instanceof, event type string, or FQCN::getType()
private static function matchesRaisedEvent($event, string $eventTypeOrClass): bool
```

### Cleanup

`$raisedEventsPerClass` is cleared in `resetAllFakes()`:

```php
public static function resetAllFakes(): void
{
    // ... existing fake cleanup ...
    self::$raisedEventsPerClass = [];
}
```

Per-class indexing prevents cross-contamination: `ActionA::runWithState()` → `ActionB::runWithState()` → `ActionA::assertRaised()` correctly reads ActionA's events.

## Issue 2: InteractsWithMachines Doesn't Reset Class-Based Spies

### Problem

`InteractsWithMachines` trait resets `Machine::$machineFakes`, `CommunicationRecorder`, and `InlineBehaviorFake` — but NOT class-based spies created via `Action::spy()` or `Guard::shouldReturn()`. Test isolation leaks.

### Current

```php
protected function tearDownInteractsWithMachines(): void
{
    Machine::resetMachineFakes();
    CommunicationRecorder::reset();
    InlineBehaviorFake::resetAll();
    // ❌ InvokableBehavior::resetAllFakes() MISSING
}
```

### Fix

Add one line:

```php
protected function tearDownInteractsWithMachines(): void
{
    Machine::resetMachineFakes();
    CommunicationRecorder::reset();
    InlineBehaviorFake::resetAll();
    InvokableBehavior::resetAllFakes();  // ADD
}
```

This also clears `$raisedEventsPerClass` from Issue 1.

## Issue 3: Machine::fake() Still Executes Parent's `with` Closure

### Problem

When `FindeksMachine::fake()` is called, `handleFakedMachineInvoke()` still calls `$invokeDefinition->resolveChildContext($state->context)` which executes the `with` closure. If the closure accesses model properties (e.g., `$ctx->farmer->birthdate`), it crashes when context doesn't have the model (because calculators were spied).

```php
// Parent definition:
'with' => fn (CarSalesContext $ctx): array => [
    'tckn' => $ctx->farmer->tckn,          // crashes if farmer is null
    'birthDate' => $ctx->farmer->birthdate,
],

// Test:
FindeksMachine::fake(finalState: 'report_saved');
// → still runs with() closure → crash
```

### Fix

In `handleFakedMachineInvoke()`, wrap `resolveChildContext()` in try-catch. If the `with` closure crashes (missing model properties), gracefully fall back to empty array:

```php
// Before:
$childContext = $invokeDefinition->resolveChildContext($state->context);

// After:
try {
    $childContext = $invokeDefinition->resolveChildContext($state->context);
} catch (\Throwable) {
    $childContext = [];
}
```

This preserves `assertChildInvokedWith` behavior when context IS available, and doesn't crash when it isn't. The `with` closure runs best-effort — if it works, the resolved context is recorded for assertions. If it fails (spied calculators left model properties null), the test still passes.

## Documentation Changes

### 1. `docs/testing/isolated-testing.md` — Add "Asserting Raised Events" Section

After the existing "Actions — modifies context" section (~line 55), add:

```markdown
### Actions — asserting raised events

Actions that call `$this->raise()` push events onto an internal queue. After `runWithState()`,
use static assertions to verify which events were raised:

\```php
CheckProtocolAction::runWithState($state);

CheckProtocolAction::assertRaised(ProtocolUndecidedEvent::class);
CheckProtocolAction::assertNotRaised(ProtocolRejectedEvent::class);
CheckProtocolAction::assertRaisedCount(1);
\```

Supports both FQCN and event type strings:

\```php
CheckProtocolAction::assertRaised('PROTOCOL_UNDECIDED');
CheckProtocolAction::assertRaised(ProtocolUndecidedEvent::class);
\```

For actions that should NOT raise any events:

\```php
StoreDataAction::runWithState($state);
StoreDataAction::assertNothingRaised();
\```

Multiple raised events — assert each individually:

\```php
MultiStepAction::runWithState($state);
MultiStepAction::assertRaised('STEP_ONE_DONE');
MultiStepAction::assertRaised('STEP_TWO_DONE');
MultiStepAction::assertRaisedCount(2);
\```
```

Update summary table (~line 328) — add:

```markdown

```
