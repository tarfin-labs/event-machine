## 3. Test Stub Design

### ScenarioTestMachine
Minimal machine that covers all 5 state classifications:

```
idle (TRANSIENT) → @always → routing (TRANSIENT, guarded)
  → [IsEligibleGuard] → processing (DELEGATION, job: ProcessJob)
     → @done → reviewing (INTERACTIVE)
        → APPROVE → approved (FINAL)
        → REJECT → rejected (FINAL)
  → [!IsEligibleGuard] → blocked (FINAL)
```

Plus a parallel section:
```
parallel_check (PARALLEL)
  → region_a: check_a → @done
  → region_b: check_b → @done
  → @done → review_complete
  → @fail → check_failed
  → SKIP_CHECK → skipped (on-transition on parallel state)
```

### ScenarioChildMachine
Simple child for delegation testing:
```
idle → @always → working (job: ChildJob) → @done → completed (FINAL)
                                          → @fail → failed (FINAL)
```
