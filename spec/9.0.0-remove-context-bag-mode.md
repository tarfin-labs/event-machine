# Shared TypedData Base + Remove Context Bag Mode + Typed Events

> Extract a shared `TypedData` base class from ContextManager. Make EventBehavior extend it — typed events become a natural consequence. Remove context bag mode. Replace `registerCast()` with config-driven cast resolution. Part of the v9.0 PR.

---

## Motivation

### Why a shared base?

ContextManager and EventBehavior are fundamentally the same thing — typed, validated, castable data containers. They share:

| Capability | ContextManager (current) | EventBehavior (current) |
|-----------|------------------------|------------------------|
| `from(array)` | Reflection-based | Hardcoded field list |
| `toArray()` | Reflection-based | Hardcoded field list |
| `rules()` / `messages()` | Laravel Validator | Laravel Validator |
| `selfValidate()` | ✅ | ✅ |
| `validateAndCreate()` | ✅ | ✅ |
| Cast resolution | 3-layer (auto, registry, explicit) | None |
| Reflection cache | ✅ | None |

The only structural difference is data wrapping:
- Context: flat (`['count' => 1]`)
- Event: wrapped (`['type' => 'X', 'payload' => ['count' => 1], 'version' => 1]`)

This difference is handled by template methods — the core reflection/cast/validation logic is identical.

### Why remove bag mode?

- **Zero usage in backend** — all 16 machines use typed context classes
- All contexts become explicitly defined — safer, more readable
- Eliminates dual-mode complexity in the engine
- v9.0 is already a major breaking change

### Why typed events?

- `$event->vehicle_id` instead of `$event->payload['vehicle_id']`
- **"Type Hint IS the Schema"** — same philosophy as typed context
- IDE autocomplete, compile-time type safety, cast support
- `rules()` uses flat keys (no `'payload.'` prefix)
- With the shared base, typed events come for free — no special dual-mode code

### Why config-driven casts?

EventMachine's principle: **look at the machine definition (or its parts) and see the complete picture.** A `registerCast()` call hidden in a service provider is invisible from the machine definition.

Replace the global registry with a 4-layer resolution:

```
Layer 1: casts()      → per-property, per-class (existing)
Layer 2: typeCasts()  → per-type, per-class (new — visible in the class)
Layer 3: config       → per-type, app-wide (visible in config/machine.php)
Layer 4: auto-detect  → built-in types (Model, Enum, DateTime, Arrayable)
```

## Architecture

### Class Hierarchy

```
TypedData (abstract)
│   from(), toArray(), rules(), selfValidate(), validateAndCreate()
│   Reflection cache, 4-layer cast resolution
│   Template methods: extractInputData(), buildArray(),
│                     isInfrastructureField(), hydrateInfrastructure()
│
├── ContextManager extends TypedData
│   + machine identity (machineId, parentMachineId, isChildMachine)
│   + get/set/has (engine dict API)
│   + computedContext(), toResponseArray()
│   Template overrides: none (default flat behavior)
│
└── EventBehavior extends TypedData
    + type, version, source, isTransactional, actor (infrastructure)
    + InteractsWithData trait
    + getType(), getScenario(), actor()
    + __get/__isset (payload key access for untyped events)
    Template overrides:
      extractInputData($data)  → $data['payload']
      buildArray($props)       → {type, payload: $props, version, ...}
      isInfrastructureField()  → true for type/payload/version/source/isTransactional
      hydrateInfrastructure()  → set type, version, source, actor from wire data
```

### Cast Resolution — 4 Layers

```php
// Layer 1: casts() — per-property, per-class
// "This specific property uses this specific cast"
public static function casts(): array
{
    return [
        'order_items' => [OrderItemData::class],    // Collection<DTO>
        'special_money' => CustomMoneyCast::class,   // override config default
    ];
}

// Layer 2: typeCasts() — per-type, per-class
// "Any property of this type in THIS class uses this cast"
public static function typeCasts(): array
{
    return [
        Money::class => MoneyCast::class,
    ];
}

// Layer 3: config/machine.php — per-type, app-wide
// "Any property of this type in ANY class uses this cast"
// Visible in config file, publishable, standard Laravel pattern
'casts' => [
    \Brick\Money\Money::class => \App\Machines\Concerns\MoneyCast::class,
],

// Layer 4: auto-detect — built-in, zero config
// Model → getKey() / ::find()
// BackedEnum → ->value / ::from()
// DateTimeInterface → ->format('c') / Carbon::parse()
// Arrayable → ->toArray() (serialize only)
```

**Resolution order:** For each property, check layers top to bottom. First match wins.

**Why `typeCasts()` over `registerCast()`?**
- Visible in the class — you see the complete picture
- No hidden service provider magic
- Per-class scope — different classes can use different casts for the same type
- Inheritable — a `BaseContext` can define `typeCasts()`, subclasses inherit

**Why config layer?**
- App-wide defaults — register once, applies everywhere
- Visible in `config/machine.php` (standard Laravel location)
- Publishable via `php artisan vendor:publish`
- `typeCasts()` can override per-class if needed

### `TypedData` Base Class

```php
// src/TypedData.php
abstract class TypedData implements Arrayable, \JsonSerializable
{
    // ── Reflection Cache ─────────────────────────────────

    private static array $parameterCache = [];
    private static array $propertyCache  = [];

    protected static function getCachedParameters(): array { /* same as current ContextManager */ }
    protected static function getCachedProperties(): array { /* same as current ContextManager */ }

    protected static function getUserProperties(): array
    {
        return array_filter(
            static::getCachedProperties(),
            fn (string $name) => !static::isInfrastructureField($name),
            ARRAY_FILTER_USE_KEY,
        );
    }

    public static function flushState(): void
    {
        self::$parameterCache = [];
        self::$propertyCache  = [];
    }

    // ── Template Methods (subclasses override) ───────────

    /** Extract user data from the raw input array. */
    protected static function extractInputData(array $data): array { return $data; }

    /** Wrap serialized user properties into the output format. */
    protected function buildArray(array $properties): array { return $properties; }

    /** Is this property name an infrastructure field (not user data)? */
    protected static function isInfrastructureField(string $name): bool { return false; }

    /** Set infrastructure fields on a freshly created instance. */
    protected static function hydrateInfrastructure(static $instance, array $data): void {}

    // ── Cast/Transform (4-layer resolution) ──────────────

    /** Layer 1: Per-property cast overrides. */
    public static function casts(): array { return []; }

    /** Layer 2: Per-type cast overrides (class-level). */
    public static function typeCasts(): array { return []; }

    /** Layer 3: App-wide type casts from config/machine.php. */
    protected static function configCasts(): array
    {
        return config('machine.casts', []);
    }

    // deserializeValue(), serializeValue(), applyCast(),
    // autoDeserialize(), extractTypeClass()
    // → same logic as current ContextManager but with 4-layer resolution:
    //   1. casts()[$name]           → per-property
    //   2. typeCasts()[$typeClass]  → per-type, per-class
    //   3. configCasts()[$typeClass] → per-type, app-wide
    //   4. auto-detect              → Model, Enum, DateTime, Arrayable

    // ── Validation ───────────────────────────────────────

    public static function rules(): array { return []; }
    public static function messages(): array { return []; }

    public function selfValidate(): void
    {
        static::performValidation($this->toArray());
    }

    public static function validateAndCreate(array|Arrayable $payload): static
    {
        $data = is_array($payload) ? $payload : $payload->toArray();
        static::performValidation($data);
        return static::from($data);
    }

    protected static function performValidation(array $data): void
    {
        $rules = static::rules();
        if ($rules === []) return;
        $validator = Validator::make($data, $rules, static::messages());
        if ($validator->fails()) {
            throw new MachineValidationException($validator);
        }
    }

    // ── Factory ──────────────────────────────────────────

    public static function from(array $data): static
    {
        $inputData = static::extractInputData($data);
        $params    = static::getCachedParameters();
        $args      = [];

        foreach ($params as $name => $param) {
            if (static::isInfrastructureField($name)) {
                continue;
            }
            if (array_key_exists($name, $inputData)) {
                $args[$name] = static::deserializeValue($name, $inputData[$name], $param);
            } elseif ($param->isDefaultValueAvailable()) {
                $args[$name] = $param->getDefaultValue();
            }
        }

        $instance = new static(...$args);
        static::hydrateInfrastructure($instance, $data);

        return $instance;
    }

    // ── Serialization ────────────────────────────────────

    public function toArray(): array
    {
        $properties = [];
        foreach (static::getUserProperties() as $name => $prop) {
            $properties[$name] = static::serializeValue($name, $prop->getValue($this));
        }
        return $this->buildArray($properties);
    }

    public function jsonSerialize(): array { return $this->toArray(); }
}
```

### `ContextManager` (extends TypedData)

```php
class ContextManager extends TypedData
{
    // No template method overrides needed — default flat behavior is correct.

    // ── Property Access (engine dict API) ────────────────

    public function get(string $key): mixed { return $this->{$key} ?? null; }

    public function set(string $key, mixed $value): mixed
    {
        $this->{$key} = $value;
        return $value;
    }

    public function has(string $key, ?string $type = null): bool
    {
        $hasKey = property_exists($this, $key);
        if (!$hasKey || $type === null) return $hasKey;
        return get_debug_type($this->get($key)) === $type;
    }

    // ── Machine Identity ─────────────────────────────────

    protected ?string $internalMachineId          = null;
    protected ?string $internalParentRootEventId  = null;
    protected ?string $internalParentMachineClass = null;

    public function setMachineIdentity(...): void { /* same as current */ }
    public function machineId(): ?string { return $this->internalMachineId; }
    public function parentMachineId(): ?string { return $this->internalParentRootEventId; }
    public function parentMachineClass(): ?string { return $this->internalParentMachineClass; }
    public function isChildMachine(): bool { return $this->internalParentRootEventId !== null; }

    // ── Computed Context ─────────────────────────────────

    protected function computedContext(): array { return []; }

    public function toResponseArray(): array
    {
        return array_merge($this->toArray(), $this->computedContext());
    }
}
```

### `EventBehavior` (extends TypedData)

```php
abstract class EventBehavior extends TypedData
{
    use InteractsWithData;

    // ── Infrastructure Fields ────────────────────────────

    public ?string $type = null;
    public ?array $payload = null;     // stored array for untyped; null for typed
    public int $version = 1;
    public SourceType $source = SourceType::EXTERNAL;
    public bool $isTransactional = true;
    private mixed $actor = null;

    private static array $infrastructure = [
        'type', 'payload', 'version', 'source', 'isTransactional',
    ];

    // ── Template Method Overrides ────────────────────────

    protected static function extractInputData(array $data): array
    {
        return $data['payload'] ?? [];
    }

    protected static function isInfrastructureField(string $name): bool
    {
        return in_array($name, self::$infrastructure, true);
    }

    protected static function hydrateInfrastructure(static $instance, array $data): void
    {
        $instance->type            = $data['type'] ?? static::getType();
        $instance->version         = $data['version'] ?? 1;
        $instance->isTransactional = $data['isTransactional'] ?? true;
        $instance->source          = isset($data['source'])
            ? ($data['source'] instanceof SourceType ? $data['source'] : SourceType::from($data['source']))
            : SourceType::EXTERNAL;
        $instance->actor = $data['actor'] ?? null;

        if (!static::hasUserProperties()) {
            $instance->payload = $data['payload'] ?? null;
        } else {
            $instance->payload = null;
        }
    }

    protected function buildArray(array $properties): array
    {
        return [
            'type'            => $this->type,
            'payload'         => $properties ?: $this->payload,
            'version'         => $this->version,
            'isTransactional' => $this->isTransactional,
            'source'          => $this->source->value,
        ];
    }

    // ── Typed/Untyped Detection ──────────────────────────

    protected static function hasUserProperties(): bool
    {
        return static::getUserProperties() !== [];
    }

    // ── Canonical Payload Accessor ───────────────────────

    /**
     * Returns the payload as an array — works in both typed and untyped mode.
     *
     * - Typed: computes from typed public properties (raw, unserialized values)
     * - Untyped: returns $this->payload (the stored array)
     *
     * Engine code and internal events should use this instead of $this->payload.
     */
    public function payload(): array
    {
        if (static::hasUserProperties()) {
            $result = [];
            foreach (static::getUserProperties() as $name => $prop) {
                $result[$name] = $prop->getValue($this);
            }
            return $result;
        }

        return $this->payload ?? [];
    }

    // ── Validation Override ──────────────────────────────

    /**
     * Always validates flat payload keys — no 'payload.' prefix needed.
     * Infrastructure fields (type, version, source) are the engine's
     * responsibility, not the event author's.
     *
     * Both typed and untyped use flat keys: 'vehicle_id', not 'payload.vehicle_id'.
     */
    public function selfValidate(): void
    {
        if (static::hasUserProperties()) {
            $serialized = [];
            foreach (static::getUserProperties() as $name => $prop) {
                $serialized[$name] = static::serializeValue($name, $prop->getValue($this));
            }
            static::performValidation($serialized);
        } else {
            static::performValidation($this->payload ?? []);
        }
    }

    public static function validateAndCreate(array $data): static
    {
        static::performValidation($data['payload'] ?? []);
        return static::from($data);
    }

    public static function stopOnFirstFailure(): bool { return true; }

    protected static function performValidation(array $data): void
    {
        $rules = static::rules();
        if ($rules === []) return;

        $validator = Validator::make($data, $rules, static::messages());
        if (static::stopOnFirstFailure()) {
            $validator->stopOnFirstFailure();
        }
        if ($validator->fails()) {
            throw new MachineEventValidationException($validator);
        }
    }

    // ── Magic Property Access ────────────────────────────

    /**
     * $event->someKey delegates to payload via __get.
     *
     * On typed events: declared properties are accessed directly (PHP skips __get).
     *   $event->vehicle_id → real property (no __get).
     *   $event->unknown_key → __get fires → data('unknown_key') → null.
     * On untyped events: all payload keys go through __get.
     */
    public function __get(string $name): mixed
    {
        return $this->data($name);
    }

    public function __isset(string $name): bool
    {
        return isset($this->payload()[$name]);
    }

    // ── Abstract & API ───────────────────────────────────

    abstract public static function getType(): string;

    public function actor(ContextManager $context): mixed { return $this->actor; }

    public function getScenario(): ?string
    {
        return $this->payload()['scenarioType'] ?? null;
    }

    public function all($keys = null): array
    {
        $input = $this->payload();
        if (!$keys) return $input;

        $results = [];
        foreach (is_array($keys) ? $keys : func_get_args() as $key) {
            Arr::set($results, $key, Arr::get($input, $key));
        }
        return $results;
    }

    public function data($key = null, $default = null): mixed
    {
        return data_get($this->all(), $key, $default);
    }

    // toArray() is NOT overridden — TypedData::toArray() calls buildArray()
    // which EventBehavior overrides to wrap properties in the wire format.

    public static function forTesting(array $attributes = []): static
    {
        return static::from(array_merge([
            'type'    => static::getType(),
            'payload' => [],
            'version' => 1,
        ], $attributes));
    }
}
```

### How Typed and Untyped Events Work

**Typed event** (has constructor properties):
```php
class VehicleSubmittedEvent extends EventBehavior
{
    public function __construct(
        public int $vehicle_id = 0,
        public string $plate_number = '',
    ) {}

    public static function getType(): string { return 'VEHICLE_SUBMITTED'; }

    public static function rules(): array
    {
        return [
            'vehicle_id'   => ['required', 'integer'],
            'plate_number' => ['required', 'string'],
        ];
    }
}

// from(['type' => 'X', 'payload' => ['vehicle_id' => 1, 'plate_number' => 'ABC']])
//   → extractInputData() returns ['vehicle_id' => 1, 'plate_number' => 'ABC']
//   → reflection maps to constructor params
//   → hydrateInfrastructure() sets type, version, source, payload=null
//
// $event->vehicle_id    → real property (IDE autocomplete, type-safe)
// $event->payload()     → computed from typed properties
// $event->payload       → null (property is null on typed events)
// $event->toArray()     → ['type' => 'X', 'payload' => ['vehicle_id' => 1, ...], ...]
```

**Untyped event** (no constructor properties):
```php
class LegacyEvent extends EventBehavior
{
    public static function getType(): string { return 'LEGACY'; }

    public static function rules(): array
    {
        return [
            'vehicle_id' => ['required', 'integer'],  // flat — same as typed
        ];
    }
}

// from(['type' => 'X', 'payload' => ['vehicle_id' => 1]])
//   → extractInputData() returns ['vehicle_id' => 1]
//   → getUserProperties() is empty → no reflection mapping
//   → hydrateInfrastructure() sets type, version, source, payload=['vehicle_id' => 1]
//
// $event->vehicle_id    → __get → data('vehicle_id') → 1
// $event->payload()     → returns $this->payload
// $event->payload       → ['vehicle_id' => 1] (stored array)
// $event->toArray()     → ['type' => 'X', 'payload' => ['vehicle_id' => 1], ...]
```

**No dual-mode branching in `from()` or `toArray()`.** The template methods handle the difference naturally. `buildArray($properties ?: $this->payload)` selects the right source automatically.

## Design Decisions

### 1. `TypedData` naming

**Decision: `TypedData`.** Descriptive of what both Context and Event are — typed data containers with reflection, validation, and cast.

### 2. Exception type in `performValidation()`

**Decision: Subclass-specific exceptions.** ContextManager overrides `performValidation()` to throw `MachineContextValidationException`. EventBehavior overrides to throw `MachineEventValidationException`. The base class provides shared validation logic.

### 3. Cast resolution — 4 layers instead of `registerCast()`

**Decision: Remove `registerCast()` API.** Replace with config-driven + class-level resolution.

| Layer | Method | Scope | Visibility |
|-------|--------|-------|------------|
| 1. `casts()` | Per-property | Per-class | In the class |
| 2. `typeCasts()` | Per-type | Per-class | In the class |
| 3. `config('machine.casts')` | Per-type | App-wide | In config/machine.php |
| 4. Auto-detect | Built-in types | Universal | In TypedData |

**Resolution order:** First match wins. `casts()` > `typeCasts()` > config > auto-detect.

**Migration from `registerCast()`:**
```php
// v8: AppServiceProvider::boot()
ContextManager::registerCast(Money::class, MoneyCast::class);

// v9 option A: config/machine.php (recommended for app-wide)
'casts' => [Money::class => MoneyCast::class],

// v9 option B: typeCasts() (for per-class override)
public static function typeCasts(): array
{
    return [Money::class => MoneyCast::class];
}
```

### 4. `$event->payload` property vs `payload()` method

**Decision: Both exist.**

- `$event->payload` — public property. Stores the array for untyped events. Null for typed events.
- `$event->payload()` — method. Canonical accessor that works in both modes.

**Why keep the property?** Backend uses `new CompletedEvent(payload: $event->payload)` — constructor named parameter. Removing it breaks this pattern.

**Engine code and internal events must use `payload()` method**, not `$event->payload` property.

### 5. Flat validation — no `'payload.'` prefix

**Decision: Mandatory for all events in v9.0.** Validation always receives flat payload — never the wrapped wire format. Infrastructure fields (`type`, `version`, `source`) are the engine's responsibility, not the event author's.

```php
// v8 (old): 'payload.vehicle_id' => ['required', 'integer']
// v9 (new): 'vehicle_id'         => ['required', 'integer']
```

Both typed and untyped events use flat keys. Backend impact: ~76 events need `'payload.'` prefix removed (mechanical find-replace).

### 6. `InteractsWithData` trait compatibility

**Decision: Works without modification.** The trait calls `$this->all()` and `$this->data()` which EventBehavior overrides. All trait methods work correctly in both typed and untyped mode.

### 7. `ContextManager::get()`/`set()`/`has()` stay

**Decision: Yes.** Engine code uses these. `remove()` is deleted — zero usage.

### 8. `__get`/`__set`/`__isset` on ContextManager

**Decision: Remove.** Typed contexts have real public properties. Only EventBehavior has `__get`/`__isset` (for untyped payload access).

### 9. Infrastructure field collision

**Decision: Documented constraint.** Event subclasses must not declare properties named `type`, `payload`, `version`, `source`, or `isTransactional`.

### 10. PHP property defaults without parent constructor

**Decision: Relies on PHP behavior.** Property initializers (`public bool $isTransactional = true`) run at object allocation time, independent of constructors.

### 11. `$event->payload` on typed events returns null

**Decision: Acceptable.** v9.0 major version. Verified: no `$event->payload = [...]` assignment exists in the codebase.

### 12. `forTesting()` in typed mode

**Decision: Works with defaults.** Typed events get constructor defaults. For specific values: `Event::forTesting(['payload' => ['vehicle_id' => 42]])`.

### 13. Internal events (`ChildMachineDoneEvent`, `ChildMachineFailEvent`)

**Decision: Update to use `payload()` method.** Stay untyped for now. `$this->payload[...]` → `$this->payload()[...]`.

## Implementation Phases

### Phase 1: Extract TypedData Base Class

1. Create `src/TypedData.php` — move reflection cache, cast resolution, validation, `from()`, `toArray()` from ContextManager
2. Add template methods: `extractInputData()`, `buildArray()`, `isInfrastructureField()`, `hydrateInfrastructure()`, `getUserProperties()`
3. Add `typeCasts()` method and `configCasts()` — 4-layer resolution
4. Update `ContextManager` to `extends TypedData` — remove duplicated logic, remove `registerCast()`/`flushState()`, keep get/set/has + machine identity + computedContext
5. Add `'casts' => []` to `config/machine.php`
6. Update existing cast tests: `registerCast()` → config or `typeCasts()`
7. Run quality gate — zero behavior change, pure refactor

### Phase 2: EventBehavior → extends TypedData

1. `EventBehavior extends TypedData` instead of standalone
2. Add template method overrides: `extractInputData()`, `buildArray()`, `isInfrastructureField()`, `hydrateInfrastructure()`
3. Add `payload()` method, `hasUserProperties()`
4. Override `selfValidate()` and `validateAndCreate()` — always validate flat payload
5. Add `__get`/`__isset` for payload key access
6. Rewrite `all()`, `data()`, `getScenario()` to use `payload()` method
7. Update `$event->payload` reads across engine: State, Machine, MachineDefinition, InvokableBehavior, MachineController
8. Update internal events: `ChildMachineDoneEvent`, `ChildMachineFailEvent` → `$this->payload()[]`
9. Update event stubs: remove `'payload.'` prefix from `rules()` keys
10. Run quality gate

### Phase 3: Remove Context Bag Mode

1. Delete `src/Context.php`
2. `ContextManager` — remove `remove()`, `__get`/`__set`/`__isset`

## Risks

| Risk | Mitigation |
|------|------------|
| TypedData extraction breaks existing behavior | Phase 1 is pure refactor — all existing tests must pass unchanged |
| ~363 test context config changes | Pattern-based: most use the same key sets, a few stub classes suffice |
| `$event->payload` returns null on typed events | v9.0 major version. Engine code updated in Phase 2. `payload()` method is canonical accessor. Constructor named param `new Event(payload: [...])` still works. |
| Typed/untyped event coexistence | No dual-mode branching — template methods and `$properties ?: $this->payload` handle it naturally |
| `InteractsWithData` trait conflicts | `all()` and `data()` are overridden — all trait methods work correctly in both modes |
| Backend `rules()` prefix removal (~76 events) | Mandatory in v9.0 — mechanical find-replace of `'payload.'` prefix |
| Backend typed event migration (146 events) | Gradual: coexistence allows one-by-one migration |
| `registerCast()` removal | Config-based replacement is standard Laravel pattern. `typeCasts()` provides class-level alternative. Migration is straightforward. |
| Config cast resolution — `config()` not available in tests | `configCasts()` is a protected method — test subclasses can override it. Also, test setup can set config via `$app['config']->set()`. |
