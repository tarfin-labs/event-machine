# Release Notes — 9.15.0 Sync-Queue Delegation & startingAt @always Drain

Delegation correctness + testing DX release. One behavioral change in the testing API (see Upgrade notes); upgrade with `composer update tarfin-labs/event-machine`.

## Fixed

- **Child-job completions are no longer silently lost on `QUEUE_CONNECTION=sync`.** `ChildJobJob` / `ChildMachineJob` / `ChildMachineTimeoutJob` / queued `ListenerJob` used to be dispatched mid-transition, before the parent's new state was persisted. On the sync driver the job ran inline, `ChildMachineCompletionJob` restored the parent at its **pre-transition** state and discarded the completion as "already transitioned" — the machine hung in the invoking state forever, with no exception, warning, or timeout. Dispatches are now buffered in `MachineDefinition::$pendingChildDispatches` and flushed by `Machine::persist()`, after the state those jobs will restore is durably written. The same ordering closes a persist-vs-worker race window on async queues.

- **Sync queue is now a fully supported mode.** `persist → child job → completion → @done/@fail routing` runs inline and correctly — after `send()` returns, the parent has already advanced past the delegating state, and the same instance stays usable (it reloads when the inline chain advanced the persisted state). Browser/e2e test suites work without Horizon or any queue worker.

- **`machine:validate` composer.json parsing** passed `JSON_THROW_ON_ERROR` as the `json_decode` depth argument; it now uses valid defaults with the graceful null-on-error path intact.

## Changed

- **`Machine::startingAt()` now drains eventless (`@always`) transitions.** A state whose `@always` guard passes is not a restable configuration — the real machine can never be observed resting there. `startingAt` still skips the target's entry actions and job dispatch, but then stabilizes exactly like a real start, making auto-routing states directly testable:

  ```php
  // Assert the @always routing itself — no artificial events needed
  ApplicationMachine::startingAt('checking_info', context: ['score' => 10])
      ->assertState('rejected');

  // Park AT the transient state deliberately: pin its guards false
  ApplicationMachine::startingAt('checking_info', guards: [IsEligibleGuard::class => false])
      ->assertState('checking_info');
  ```

  `Machine::assertTransitions()` table rows inherit the drain.

## Upgrade notes

- Tests that relied on `startingAt()` parking at a transient state (one with a passing `@always` guard) will now observe the routed state. Either assert the routed state (usually what the test meant) or pin the `@always` guards false via the `guards:` parameter to keep parking.
- Workarounds that sent artificial events to trigger `@always` routing after `startingAt()` can be removed.
