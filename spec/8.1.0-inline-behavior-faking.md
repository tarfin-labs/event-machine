# Plan: Inline Behavior Faking

**Status:** Ready for Implementation
**Created:** 2026-03-12
**Last Updated:** 2026-03-12 (v2 — post-review)
**Scope:** Make inline closures (actions, guards, calculators) fakeable through the existing TestMachine API.

---

## Design Philosophy

Unlike XState v5 (which intentionally makes inline closures untestable to nudge users toward named actions) and Symfony Workflow (which avoids inline closures entirely), EventMachine provides **full first-class support for inline behavior testing**.

Inline closures are a core feature of EventMachine — they enable rapid prototyping, simple one-liner behaviors, and compact machine definitions. Forcing users to create class files for every trivial behavior adds unnecessary ceremony. Our testing infrastructure should support both definition styles equally.

---

## Problem Statement

EventMachine supports two behavior definition styles:

1. **Class-based** — FQCN extending `InvokableBehavior` (e.g., `SendEmailAction::class`)
2. **Inline** — Closures in the behavior array (e.g., `'myAction' => fn(State $state) => ...`)

Class-based behaviors are resolved via `App::make()` and are fully fakeable through the `Fakeable` trait (which registers `App::bind()` overrides). Inline closures bypass the container entirely — `getInvokableBehavior()` returns them directly from the behavior array. This makes them **invisible to the fake/spy mechanism**.

### Real-World Impact

When a developer writes an inline action (e.g., `'broadcastAction' => fn(State $state) => SomeEvent::dispatch(...)`) and wants to test it, the only option is to copy the same closure into a test machine's behavior array. This creates a circular test: the test defines the behavior, triggers it, and asserts it ran — testing its own setup, not the production code. The developer's analogy: *"It's like setting a property, then in the test setting the same property, then asking if I set it."*

### Technical Root Cause

`MachineDefinition::getInvokableBehavior()` (line 599–619) has three resolution paths:

```
behaviorDefinition string
  ├─ is_subclass_of(InvokableBehavior)? → App::make($class)     ← FAKEABLE via container
  └─ behavior array lookup
       ├─ callable?     → return as-is                           ← NOT FAKEABLE
       └─ FQCN string?  → App::make($class)                     ← FAKEABLE via container
```

Path 2 (callable) returns the closure directly. No container involvement, no interception point.

---

## Why Not Wrap at Resolution Time?

The natural instinct is to intercept at `getInvokableBehavior()` — replace the closure before it's returned. This fails because of parameter injection.

`InvokableBehavior::injectInvokableBehaviorParameters()` (line 140–175) uses `ReflectionFunction` on the callable to inspect its parameter types and inject the correct values (`State`, `ContextManager`, `EventBehavior`, etc.). If we replace the closure with a different one (e.g., `fn() => null`), the reflection reads the **replacement's** signature, not the original's, and injects wrong parameters — or throws `ArgumentCountError`.

---

## Solution: Invocation-Site Interception

Instead of replacing the callable, we intercept at the point where the callable is **invoked**. Parameter injection still uses the original closure's reflection. The interception checks a static registry and either:

- **Spy mode**: Records the call, then runs the original
- **Fake mode**: Records the call, runs a replacement (default: variadic no-op), skips original

### Invocation Sites (Verified)

| # | Location | Method | What |
|---|----------|--------|------|
| 1 | `MachineDefinition.php:1811` | `runAction()` | All actions (transition, entry, exit) |
| 2 | `TransitionDefinition.php:183` | `getFirstValidTransitionBranch()` | All guards |
| 3 | `TransitionDefinition.php:259` | `runCalculators()` | All calculators |
| 4 | `Machine.php:702` | `result()` | Result behaviors (deferred) |

**Entry/exit coverage:** `StateDefinition::runEntryActions()` (line 538–559) and `runExitActions()` (line 510–531) both call `$this->machine->runAction()`. They are covered by Site 1.

**`@done` / `@fail` coverage:** These handlers resolve transitions via `getFirstValidTransitionBranch()` (Site 2 for guards) and execute actions via `runAction()` (Site 1). Covered automatically.

**`@always` coverage:** Always transitions flow through `getFirstValidTransitionBranch()` (Site 2). Covered.

**Colon-stripping:** At all three sites, parametric behavior keys (e.g., `'myAction:arg1,arg2'`) are split via `explode(':', ..., 2)` **before** the intercept point. The `$actionDefinition` / `$guardDefinition` / `$calculatorDefinition` variable at the intercept line contains the **bare key** (e.g., `'myAction'`), matching what the developer registers. Verified at:
- `MachineDefinition.php:1772` (actions)
- `TransitionDefinition.php:160` (guards)
- `TransitionDefinition.php:241` (calculators)

**Result behaviors (Site 4, deferred):** Result behaviors have a different invocation pattern — they call the callable directly with explicit arguments (`$context, $eventBehavior, $arguments`) rather than using `injectInvokableBehaviorParameters()`. Intercept would need a different approach. Deferred to a follow-up iteration since result behaviors are rarely used as inline closures.

## Implementation

### Phase 1: `InlineBehaviorFake` Class

**File:** `src/Testing/InlineBehaviorFake.php`

**Autoload note:** `src/` is in the production `autoload` (not `autoload-dev`) in `composer.json`. This is intentional — `src/Testing/TestMachine.php` follows the same convention. Consumers need these classes to test their machines.

```php
<?php

declare(strict_types=1);

namespace Tarfinlabs\EventMachine\Testing;

use Closure;
use PHPUnit\Framework\Assert;
use Tarfinlabs\EventMachine\Behavior\InvokableBehavior;

class InlineBehaviorFake
{
    /**
     * Registered inline behavior fakes.
     *
     * null value = spy (run original, record calls)
     * Closure value = fake (skip original, run replacement)
     *
     * @var array<string, Closure|null>
     */
    private static array $fakes = [];

    /** @var array<string, list<array>> Recorded invocations per key */
    private static array $calls = [];

    // ─── Registration ─────────────────────────────

    /**
     * Spy mode: record calls AND run the original closure.
     *
     * @throws \InvalidArgumentException If $key is a class-based behavior FQCN
     */
    public static function spy(string $key): void
    {
        self::guardAgainstFqcn($key);

        self::$fakes[$key] = null;
        self::$calls[$key] = [];
    }

    /**
     * Fake mode: record calls, skip original, run replacement (default: variadic no-op).
     *
     * IMPORTANT: The replacement receives the same positional parameters that the
     * engine would inject into the original closure (based on the original's signature
     * via ReflectionFunction). The default no-op uses variadic args to safely absorb
     * any parameter combination. Custom replacements must accept variadic args or
     * match the original closure's parameter signature.
     *
     * @throws \InvalidArgumentException If $key is a class-based behavior FQCN
     */
    public static function fake(string $key, ?Closure $replacement = null): void
    {
        self::guardAgainstFqcn($key);

        self::$fakes[$key] = $replacement ?? static fn (mixed ...$args) => null;
        self::$calls[$key] = [];
    }

    /**
     * Fake a guard/calculator with a specific return value.
     *
     * Convenience method for the common pattern of faking a guard to return true/false.
     */
    public static function shouldReturn(string $key, mixed $value): void
    {
        self::fake($key, static fn (mixed ...$args) => $value);
    }

    // ─── Interception ─────────────────────────────

    /**
     * Called at invocation sites. Records the call if tracked.
     *
     * @param  string  $key  The bare behavior key (colon-stripped by the engine before this call)
     * @param  array  $parameters  The positional parameters injected by injectInvokableBehaviorParameters()
     *
     * @return bool True if the original should be SKIPPED (fake mode).
     */
    public static function intercept(string $key, array $parameters): bool
    {
        if (!array_key_exists($key, self::$fakes)) {
            return false;
        }

        self::$calls[$key][] = $parameters;

        // null = spy (don't skip), Closure = fake (skip original)
        return self::$fakes[$key] !== null;
    }

    /**
     * Get the replacement closure for a faked behavior.
     */
    public static function getReplacement(string $key): ?Closure
    {
        return self::$fakes[$key];
    }

    // ─── Inspection ───────────────────────────────

    public static function isFaked(string $key): bool
    {
        return array_key_exists($key, self::$fakes);
    }

    /** @return list<array> All recorded invocations for the key */
    public static function getCalls(string $key): array
    {
        return self::$calls[$key] ?? [];
    }

    // ─── Assertions ───────────────────────────────

    public static function assertRan(string $key): void
    {
        Assert::assertArrayHasKey($key, self::$fakes,
            "Inline behavior '{$key}' was not faked or spied. Call InlineBehaviorFake::spy('{$key}') or faking(['{$key}']) first."
        );
        Assert::assertNotEmpty(self::$calls[$key],
            "Expected inline behavior '{$key}' to have been called, but it was not."
        );
    }

    public static function assertNotRan(string $key): void
    {
        Assert::assertArrayHasKey($key, self::$fakes,
            "Inline behavior '{$key}' was not faked or spied."
        );
        Assert::assertEmpty(self::$calls[$key],
            "Expected inline behavior '{$key}' NOT to have been called, but it was called ".count(self::$calls[$key]).' time(s).'
        );
    }

    public static function assertRanTimes(string $key, int $expected): void
    {
        Assert::assertArrayHasKey($key, self::$fakes,
            "Inline behavior '{$key}' was not faked or spied."
        );
        Assert::assertCount($expected, self::$calls[$key],
            "Expected inline behavior '{$key}' to run {$expected} time(s), ran ".count(self::$calls[$key]).'.'
        );
    }

    /**
     * Assert the behavior was called with parameters matching the callback.
     *
     * The callback receives the FULL injected parameter array as a single argument
     * (not spread). This avoids ArgumentCountError since the parameter count varies
     * per closure signature.
     *
     * Usage:
     *   InlineBehaviorFake::assertRanWith('myAction', function (array $params) {
     *       [$state] = $params;  // destructure based on the original closure's signature
     *       return $state->context->get('total') === 100;
     *   });
     */
    public static function assertRanWith(string $key, Closure $callback): void
    {
        Assert::assertArrayHasKey($key, self::$fakes,
            "Inline behavior '{$key}' was not faked or spied."
        );

        $matched = false;
        foreach (self::$calls[$key] as $callArgs) {
            if ($callback($callArgs) === true) {
                $matched = true;
                break;
            }
        }

        Assert::assertTrue($matched,
            "Expected inline behavior '{$key}' to have been called with matching arguments."
        );
    }

    // ─── Cleanup ──────────────────────────────────

    public static function reset(string $key): void
    {
        unset(self::$fakes[$key], self::$calls[$key]);
    }

    public static function resetAll(): void
    {
        self::$fakes = [];
        self::$calls = [];
    }

    // ─── Internal ─────────────────────────────────

    /**
     * Prevent accidental registration of class-based behavior FQCNs.
     *
     * If a developer calls InlineBehaviorFake::fake('App\Actions\SendEmail'),
     * this would silently register but never intercept (the engine resolves FQCNs
     * through the container, not through inline lookup). The error message guides
     * toward the correct API.
     */
    private static function guardAgainstFqcn(string $key): void
    {
        if (class_exists($key) && is_subclass_of($key, InvokableBehavior::class)) {
            throw new \InvalidArgumentException(
                "'{$key}' is a class-based behavior (InvokableBehavior subclass). "
                ."Use {$key}::spy() or {$key}::fake() instead of InlineBehaviorFake."
            );
        }
    }
}
```

#### Key Design Decisions

1. **Variadic no-op:** Default replacement is `fn(mixed ...$args) => null`, NOT `fn() => null`. The engine calls `($replacement)(...$injectedParams)`, so the replacement must accept the same positional arguments. A zero-argument closure would throw `ArgumentCountError`. Variadic `mixed` safely absorbs any parameter combination.

2. **PHPUnit Assert, not Pest expect():** Using `PHPUnit\Framework\Assert` keeps the class framework-agnostic (works in both PHPUnit and Pest contexts). The `Fakeable` trait uses Mockery assertions for the same reason — framework independence.

3. **Static registry:** Matches the `Fakeable` trait's `static $fakes` pattern. Both are process-scoped and cleaned up in `afterEach`.

4. **FQCN guard:** `fake()` and `spy()` reject class-based behavior FQCNs with an actionable error message. This prevents silent misuse where a developer registers a FQCN in `InlineBehaviorFake` but the engine resolves it through the container — the fake would never intercept, and `assertRan()` would confusingly fail.

5. **`assertRanWith` receives array, not spread:** The callback in `assertRanWith()` receives the full parameter array as a **single argument** (not spread). This avoids `ArgumentCountError` since the injected parameter count varies per closure signature. The developer destructures what they need: `fn(array $params) => $params[0]->context->get('x') === 1`. This differs from the class-based `assertRanWith` (which spreads via Mockery's `withArgs`), but is necessary because inline closures have heterogeneous signatures.

### Phase 2: Engine Changes (3 Invocation Sites)

#### Site 1: `MachineDefinition::runAction()` (line 1811)

This covers ALL action invocations: transition actions, entry actions, and exit actions.

**Note:** The existing variable name `$actionBehaviorParemeters` contains a typo ("Paremeters" instead of "Parameters"). The intercept code must use this same misspelled name to match the existing code. A separate follow-up can fix the typo across the codebase.

```php
// BEFORE (line 1811):
($actionBehavior)(...$actionBehaviorParemeters);

// AFTER:
if (InlineBehaviorFake::intercept($actionDefinition, $actionBehaviorParemeters)) {
    $replacement = InlineBehaviorFake::getReplacement($actionDefinition);
    ($replacement)(...$actionBehaviorParemeters);
} else {
    ($actionBehavior)(...$actionBehaviorParemeters);
}
```

**History events:** `ACTION_START` (line 1789) is recorded BEFORE the intercept point. `ACTION_FINISH` (line 1833) is recorded AFTER. Both are preserved regardless of whether the behavior is faked — maintaining the history contract. The only difference is that the faked behavior's body doesn't execute; the engine still sees a complete action lifecycle.

**Event queue check:** Lines 1813–1827 check if the action raised new events. When faked with the default no-op, no events are raised and this block is a no-op. When faked with a custom replacement that calls `$this->raise()`, this would not work because the replacement closure doesn't have access to the event queue. This is acceptable: inline closures don't have `$this->raise()` access either (they're not InvokableBehavior instances).

**Context validation:** Line 1830 (`$state->context->selfValidate()`) still runs after a faked action. This is correct — if a replacement modifies context, validation should still apply.

#### Site 2: `TransitionDefinition::getFirstValidTransitionBranch()` (line 183)

```php
// BEFORE (line 183):
$guardResult = ($guardBehavior)(...$guardBehaviorParameters);

// AFTER:
if (InlineBehaviorFake::intercept($guardDefinition, $guardBehaviorParameters)) {
    $replacement = InlineBehaviorFake::getReplacement($guardDefinition);
    $guardResult = ($replacement)(...$guardBehaviorParameters);
} else {
    $guardResult = ($guardBehavior)(...$guardBehaviorParameters);
}
```

**Guard return value semantics:** When faked with `InlineBehaviorFake::fake('myGuard')`, the default replacement returns `null`. For guards, `null !== false` means the guard **silently passes**. This mirrors the existing class-based behavior documented in `fakeable-behaviors.md` lines 91–99 (warning box: "Guards need shouldReturn() after faking()"). Users must call `InlineBehaviorFake::shouldReturn('myGuard', false)` to block.

The `faking()` API supports this with the key-value syntax (see Phase 3 below).

**History events:** `GUARD_PASS` / `GUARD_FAIL` events (lines 200–215) are recorded AFTER invocation. They correctly reflect the replacement's return value — if `shouldReturn('guard', false)` was set, `GUARD_FAIL` is recorded.

#### Site 3: `TransitionDefinition::runCalculators()` (line 259)

```php
// BEFORE (line 259):
($calculatorBehavior)(...$calculatorParameters);

// AFTER:
if (InlineBehaviorFake::intercept($calculatorDefinition, $calculatorParameters)) {
    $replacement = InlineBehaviorFake::getReplacement($calculatorDefinition);
    ($replacement)(...$calculatorParameters);
} else {
    ($calculatorBehavior)(...$calculatorParameters);
}
```

**Calculator pass/fail:** Lines 261–275 handle `CALCULATOR_PASS` / `CALCULATOR_FAIL`. The try/catch wraps the invocation. The replacement runs inside the try block, so if a custom replacement throws, `CALCULATOR_FAIL` is correctly recorded.

### Phase 3: TestMachine — Unified API

#### Updated `faking()` Method

The `faking()` method handles four cases in a single array:

```php
->faking([
    SendEmailAction::class,                        // numeric key, FQCN → class-based spy
    'broadcastAction',                              // numeric key, non-FQCN string → inline fake (no-op)
    'isValidGuard' => false,                        // string key, scalar value → inline fake with return value
    'calculateTaxAction' => fn(mixed ...$a) => 0,   // string key, Closure → inline fake with custom replacement
])
```

Detection logic:

```php
public function faking(array $behaviors): self
{
    foreach ($behaviors as $key => $value) {
        if (is_int($key)) {
            // Numeric key: $value is the behavior identifier
            if (is_string($value) && is_subclass_of($value, InvokableBehavior::class)) {
                // Class-based behavior → existing spy mechanism
                $value::spy();
                $this->fakedBehaviors[] = $value;
            } elseif (is_string($value)) {
                // Inline behavior key → fake with no-op
                $this->validateInlineBehaviorKey($value);
                InlineBehaviorFake::fake($value);
                $this->fakedInlineBehaviors[] = $value;
            }
        } else {
            // String key: inline behavior with specific return value or replacement
            $this->validateInlineBehaviorKey($key);

            if ($value instanceof \Closure) {
                // Custom replacement closure
                InlineBehaviorFake::fake($key, $value);
            } else {
                // Scalar return value (most common: guard true/false)
                InlineBehaviorFake::shouldReturn($key, $value);
            }

            $this->fakedInlineBehaviors[] = $key;
        }
    }

    return $this;
}
```

This design:
- Maintains backward compatibility (numeric-indexed FQCN strings work exactly as before)
- Provides ergonomic guard return value control in a single call
- Supports custom replacement closures via key-value syntax
- Matches the existing `Bus::fake([...])` / `Event::fake([...])` Laravel convention

#### Key Validation Helper

```php
/**
 * Validate that an inline behavior key exists in the machine's behavior array.
 *
 * Provides fail-fast typo detection similar to how PHP class loading catches FQCN typos.
 * Only available through TestMachine::faking() (requires machine context).
 *
 * @throws \InvalidArgumentException If the key is not found
 */
private function validateInlineBehaviorKey(string $key): void
{
    $allBehaviorKeys = array_merge(
        array_keys($this->machine->definition->behavior['actions'] ?? []),
        array_keys($this->machine->definition->behavior['guards'] ?? []),
        array_keys($this->machine->definition->behavior['calculators'] ?? []),
    );

    if (!in_array($key, $allBehaviorKeys, true)) {
        throw new \InvalidArgumentException(
            "Inline behavior key '{$key}' not found in machine definition. "
            ."Available keys: [".implode(', ', $allBehaviorKeys).']'
        );
    }
}
```

#### Updated Assertion Methods

All assertion methods dispatch based on whether the identifier is a class or a string key:

```php
public function assertBehaviorRan(string $classOrKey): self
{
    if (is_subclass_of($classOrKey, InvokableBehavior::class)) {
        $classOrKey::assertRan();
    } else {
        InlineBehaviorFake::assertRan($classOrKey);
    }

    return $this;
}

public function assertBehaviorNotRan(string $classOrKey): self
{
    if (is_subclass_of($classOrKey, InvokableBehavior::class)) {
        $classOrKey::assertNotRan();
    } else {
        InlineBehaviorFake::assertNotRan($classOrKey);
    }

    return $this;
}

public function assertBehaviorRanTimes(string $classOrKey, int $times): self
{
    if (is_subclass_of($classOrKey, InvokableBehavior::class)) {
        $classOrKey::assertRanTimes($times);
    } else {
        InlineBehaviorFake::assertRanTimes($classOrKey, $times);
    }

    return $this;
}

/**
 * Assert behavior was called with matching arguments.
 *
 * For class-based behaviors: callback receives individual Mockery-recorded args.
 * For inline behaviors: callback receives the full injected parameter array as
 * a single argument (see InlineBehaviorFake::assertRanWith() docblock).
 */
public function assertBehaviorRanWith(string $classOrKey, \Closure $callback): self
{
    if (is_subclass_of($classOrKey, InvokableBehavior::class)) {
        $classOrKey::assertRanWith($callback);
    } else {
        InlineBehaviorFake::assertRanWith($classOrKey, $callback);
    }

    return $this;
}
```

Add immediately after the existing `$fakedBehaviors` property declaration:

#### Updated `resetFakes()`

```php
public function resetFakes(): self
{
    foreach ($this->fakedBehaviors as $behavior) {
        $behavior::resetFakes();
    }
    $this->fakedBehaviors = [];

    foreach ($this->fakedInlineBehaviors as $key) {
        InlineBehaviorFake::reset($key);
    }
    $this->fakedInlineBehaviors = [];

    return $this;
}
```

### Phase 4: Global Cleanup Integration

The existing recommended `afterEach` pattern calls `IncrementAction::resetAllFakes()` (which resolves to `InvokableBehavior::resetAllFakes()`). This must also clear inline fakes to prevent test pollution.

**Change in `Fakeable::resetAllFakes()`:**

```php
public static function resetAllFakes(): void
{
    foreach (static::$fakes as $class => $mock) {
        app()->offsetUnset($class);

        foreach (array_keys($mock->mockery_getExpectations()) as $method) {
            $mock->mockery_setExpectationsFor(
                $method,
                new Mockery\ExpectationDirector($method, $mock),
            );
        }
        $mock->mockery_teardown();
    }

    static::$fakes = [];
    static::$spies = [];

    // Also reset inline behavior fakes
    \Tarfinlabs\EventMachine\Testing\InlineBehaviorFake::resetAll();
}
```

**No `class_exists()` guard needed:** `src/Testing/` is in the production `autoload` (verified in `composer.json` — `"Tarfinlabs\\EventMachine\\": "src"`), same as `TestMachine.php`. The class is always available.

This ensures the single `afterEach(fn() => IncrementAction::resetAllFakes())` pattern covers both class-based and inline fakes without requiring users to update their test setup.

## API Summary

### User-Facing API

```php
// ────────────────────────────────────────────────
// Via TestMachine (recommended)
// ────────────────────────────────────────────────

// Fake inline actions (skip original, record calls)
OrderMachine::test()
    ->faking([
        'sendEmailAction',                             // inline action → no-op
        'isValidGuard' => true,                        // inline guard → return true
        'calculateTax' => fn(mixed ...$a) => null,     // inline calculator → custom replacement
        SendEmailAction::class,                        // class-based → existing spy
    ])
    ->send('SUBMIT')
    ->assertBehaviorRan('sendEmailAction')
    ->assertBehaviorNotRan('someOtherAction')
    ->assertBehaviorRanTimes('sendEmailAction', 1)
    ->assertBehaviorRanWith('sendEmailAction', fn(array $params) => $params[0]->context->get('sent') === true);

// ────────────────────────────────────────────────
// Direct API (for advanced use cases)
// ────────────────────────────────────────────────

// Spy mode: record calls but still run original
InlineBehaviorFake::spy('broadcastAction');

// Fake mode: skip original, run no-op
InlineBehaviorFake::fake('broadcastAction');

// Fake with custom replacement (must accept variadic or matching signature)
InlineBehaviorFake::fake('calculateTaxAction', fn(ContextManager $ctx) => $ctx->set('tax', 0));

// Fake a guard with specific return value
InlineBehaviorFake::shouldReturn('isValidGuard', false);

// Assertions
InlineBehaviorFake::assertRan('broadcastAction');
InlineBehaviorFake::assertNotRan('someAction');
InlineBehaviorFake::assertRanTimes('broadcastAction', 2);
InlineBehaviorFake::assertRanWith('storeAction', fn(array $params) => $params[0]->context->get('stored') === true);

// Inspection
InlineBehaviorFake::isFaked('broadcastAction');   // bool
InlineBehaviorFake::getCalls('broadcastAction');   // list<array>

// Cleanup
InlineBehaviorFake::reset('broadcastAction');   // single key
InlineBehaviorFake::resetAll();                  // all inline fakes
```

### Semantic Comparison: fake() vs spy()

| Mode | Original runs? | Calls recorded? | Default return | Use case |
|------|---------------|-----------------|----------------|----------|
| `fake($key)` | No (replacement runs) | Yes | `null` | Skip side effects, test flow |
| `spy($key)` | Yes | Yes | Original's return | Run real code, assert it was called |
| `shouldReturn($key, $val)` | No | Yes | `$val` | Control guard/calculator returns |
| `faking([...])` (TestMachine) | No | Yes | `null` | Selective faking (matches class-based `faking()`) |

### `assertRanWith` Callback Signature Difference

| Behavior type | Callback receives | Example |
|---|---|---|
| Class-based | Individual Mockery-recorded args | `fn(ContextManager $ctx, ?EventBehavior $e) => ...` |
| Inline | Single array of injected params | `fn(array $params) => $params[0]->context->get('x')` |

The inline version passes the full parameter array as one argument to avoid `ArgumentCountError` (since the injected parameter count varies per closure signature).

## Parallel Region Considerations

Inline fakes use a static registry (`InlineBehaviorFake::$fakes`). This is process-scoped.

| Parallel Strategy | Same Process? | Inline Fakes Work? |
|-------------------|--------------|---------------------|
| `Queue::fake()` | Jobs not executed | N/A (behaviors don't run in jobs) |
| `config(['queue.default' => 'sync'])` | Yes | Yes |
| `withoutParallelDispatch()` | Yes | Yes |
| Real queue driver (production) | No | No (not a test scenario) |

**Conclusion:** Inline fakes are reliable in all test scenarios. The only failure case is real queue dispatch across processes, which is not a testing pattern.

## Backward Compatibility

| Concern | Risk | Mitigation |
|---------|------|------------|
| Existing class-based `faking()` calls | None | `is_subclass_of` check routes to existing path; numeric-indexed FQCN strings behave identically |
| Existing `resetAllFakes()` pattern | None | Updated to also call `InlineBehaviorFake::resetAll()` |
| Engine invocation performance | Minimal | `array_key_exists` on an empty array is ~5ns per call |
| Inline behaviors not tracked | None | `intercept()` returns `false` for unregistered keys → original runs unchanged |
| Class-based FQCN passed to InlineBehaviorFake | Caught | `guardAgainstFqcn()` throws `InvalidArgumentException` with actionable message |

## Open Questions / Future Work

1. **Result behaviors:** Site 4 (`Machine::result()`) is deferred. Result behaviors have a different invocation pattern and are rarely inline closures. Can be added in a follow-up if demand exists.

2. **`$actionBehaviorParemeters` typo:** The existing codebase has this misspelling in `MachineDefinition::runAction()`. The intercept code preserves it for compatibility. A separate follow-up PR should fix the typo across the codebase.

3. **Standalone use without TestMachine:** `InlineBehaviorFake` can be used directly (without TestMachine). The key validation (Phase 3) only works through `TestMachine::faking()` because it requires machine context. Direct `InlineBehaviorFake::fake('typo')` would register silently but `assertRan()` would later fail — technically correct but the error message won't mention a possible typo.

4. **Pre-existing bug in `Machine::result()`:** During review, an unrelated issue was found at `Machine.php:694–706` — `$arguments` is used with `?? null` but may be an undefined variable (not assigned when no colon is present). This should be fixed in a separate PR.
