# Inline Testing Sections — Every Feature Page Gets a Testing Section

**Status:** Draft v3 (final review applied)
**Author:** Claude
**Date:** 2025-03-18
**Scope:** Add consistent `## Testing` sections to all feature documentation pages

---

## Problem

EventMachine docs have inconsistent testing guidance across feature pages:

- **7 pages** have complete testing sections (behaviors core)
- **10 pages** have testing content but are missing `::: tip` links, need format fix, or need expansion
- **15 pages** with testable features have no testing content at all
- **15 pages** are conceptual/overview and only need a `::: tip` link
- **3 pages** are not applicable (conventions, compression, upgrading)

This inconsistency means users learning async delegation get no testing guidance, while users learning guards get comprehensive examples. Laravel docs set the standard — every feature page (Queues, Notifications, Events, Mail) ends with a Testing section.

---

## Standard: The Behaviors Pattern

The `behaviors/*.md` pages already implement the ideal pattern. Every testing section MUST follow this template:

```markdown
## Testing {Feature}

{1-2 sentence intro connecting the feature to testing}

<!-- doctest-attr: ignore -->
\```php
// Example 1: Most common test pattern
{concise, copy-pasteable code}
\```

<!-- doctest-attr: ignore -->
\```php
// Example 2: Alternative approach (if applicable)
{concise, copy-pasteable code}
\```

::: tip Full Testing Guide
For {detailed topic}, see [{Page Name}](/testing/{page}).
For {another topic}, see [{Page Name}](/testing/{page}).
:::
```

**Rules:**
1. **Keep it focused** — only test patterns relevant to THIS feature
2. **Code-first** — show the test, then explain
3. **Link, don't duplicate** — if `testing/*` has the full guide, show the basic pattern and link
4. **Existing content stays** — never remove or rewrite working testing sections (behaviors pages are the gold standard)

## Quality Checklists

### Full Section Checklist (Phase 2, 3, 4)

- [ ] `## Testing {Feature}` heading exists
- [ ] At least 1 code example (copy-pasteable)
- [ ] `::: tip Full Testing Guide` with link to relevant `testing/*` page
- [ ] Code blocks have appropriate doctest attribute (`ignore` or `no_run`)
- [ ] No duplication of content already in `testing/*` pages
- [ ] Existing tip/warning blocks preserved (not removed)
- [ ] Run `vendor/bin/doctest docs/{path} -v` — 0 failures

### Tip-Only Checklist (Phase 1, 5)

- [ ] `::: tip` block with relevant testing page link
- [ ] Link target exists and is correct
- [ ] No code examples (just the link)
