## Tespit Edilen 5 Test Pattern'i

### Pattern 1: `Machine::getGuard/getAction/getCalculator` + Direct Invocation
**Kullanım:** OrderCancellationMachine (49 test), bazı diğerleri
**Nasıl çalışır:**
```php
$guard = OrderCancellationMachine::getGuard('orderIsNotDemoGuard');
$result = $guard($context);
assertTrue($result);
```
**Avantaj:** Behavior'ı machine definition'dan çıkarıp izole test eder. Inline closure behavior'lar için tek yol.
**Eksik:** Machine-level state flow hiç test edilmiyor. Guard sıralaması, transition routing yok.

### Pattern 2: `BehaviorClass::run($context, $event)` — Static Invocation
**Kullanım:** ApplicationMachine action/guard testleri (çoğunluk)
**Nasıl çalışır:**
```php
ApprovedTransitionAction::run($context, new EventDefinition(payload: [...]));
```
**Avantaj:** Container resolution'dan geçiyor, DI çalışıyor.
**Eksik:** `run()` eventQueue geçmiyor (v5'te düzeltildi → `runWithState()`). State bağlamı yok.

### Pattern 3: `new BehaviorClass()($context, $event)` — Direct Instantiation
**Kullanım:** Bazı guard testleri, özellikle `validateRequiredContext` kontrolü
**Nasıl çalışır:**
```php
$guard = new IsFraudDetectedGuard();
$guard->validateRequiredContext($context);
$result = $guard(context: $context, event: new EventDefinition());
```
**Avantaj:** Constructor kontrolü, requiredContext doğrulaması.
**Eksik:** Container'dan geçmiyor, DI yok.

### Pattern 4: `Fakeable` Orchestration — Mock Dependencies
**Kullanım:** Transition action'lar (ApprovedTransitionAction, CancelledTransitionAction)
**Nasıl çalışır:**
```php
RecalculateForInterestRateAction::shouldRun()->once();
ApplyDiscountsAction::shouldRun()->once();
ApproveAction::shouldRun()->once();
GuarantorAction::shouldRun()->never();

ApprovedTransitionAction::run($context, $event);
// Mockery::close() doğrular
```
**Avantaj:** Orchestration action'ın hangi alt-action'ları çağırdığını test eder.
**Eksik:** Gerçek alt-action'ların sonuçlarını test etmez.

### Pattern 5: `Machine::create() → send()` — Full Integration
**Kullanım:** YALNIZCA ConversionMachine (should_persist: false)
**Nasıl çalışır:**
```php
$machine = ConversionMachine::create();
$machine->state->context->set('application', $order->application);
$state = $machine->send(event: $event);
assertEquals([...], $state->context->discountData);
```
**Neden sadece bu?** Persist etmiyor → DB mock gerekmez. Guard zinciri uzun → sıralama hatalarını yakalamak için bütünsel test gerekli.

## Kritik Eksiklikler (TestMachine İle Doldurulabilir)

### Eksiklik 1: End-to-End State Flow Testleri — HİÇ YOK (persist eden makinelerde)

**Problem:** 14 makinenin 13'ünde (Conversion hariç) `Machine::create() → send() → assertState()` testi yok.

**Neden yok?** TestMachine API'si yoktu, `should_persist: true` makinelerde her `send()` DB'ye yazmak istiyor, test ortamında bunu kontrol etmek zordu.

**TestMachine çözümü:**
```php
OrderCancellationMachine::test()
    ->withoutPersistence()
    ->send(['type' => 'START', 'payload' => ['order' => $order->id]])
    ->assertState('done.with_notifications');
```

**Etki:** OrderCancellationMachine'in `check.order → check.day → after.7Days → done.with_cancellation` tam akışı test edilebilir. Şu an yalnızca her behavior ayrı ayrı test ediliyor, routing logic (hangi günde hangi state'e gidilir) test edilmiyor.

### Eksiklik 2: Guard Sıralama ve Öncelik Testleri — YOK

**Problem:** `check.order` state'inde 5 guard ardışık çalışır. Sıralama önemli — ilk false dönen guard transition'ı engeller. Ama hiçbir test "guard sırası" doğrulamıyor.

**TestMachine çözümü:**
```php
OrderCancellationMachine::test()
    ->withoutPersistence()
    ->send(['type' => 'START', 'payload' => ['order' => $demoOrder->id]])
    ->assertGuarded('START')  // demo order → ilk guard fail
    ->assertState('fail.not_applicable');
```

### Eksiklik 3: Scenario Path Testleri — YOK

**Problem:** ApplicationMachine'in 9 senaryosu var. Hiçbiri machine-level test edilmiyor. Senaryo override'ların doğru state'lere yönlendirdiği test edilmiyor.

**TestMachine çözümü:**
```php
ApplicationMachine::test()
    ->withoutPersistence()
    ->send(ApplicationStartedEvent::forTesting([...]))
    ->assertState('score.checking');
```

### Eksiklik 4: Compound/Nested State Navigation Testleri — YOK

**Problem:** `findeks.checking → findeks.validationWaiting → ... → findeks.reportSaved` zinciri hiç test edilmiyor. 6 alt-state arası geçişler yalnızca guard/action bazında test ediliyor.

### Eksiklik 5: Global Transition Testleri — YOK

**Problem:** ApplicationMachine'de `TerminatedEvent` ve `ExpiredEvent` herhangi bir state'ten `terminated`/`expired` final state'lerine geçiş yapar. Bu "escape transition" pattern'i hiç test edilmiyor.

### Eksiklik 6: Parallel State Testleri — TAMAMEN YOK

**Problem:** CarSalesMachine paralel state kullanıyor ama sıfır test var. `@done`, `@fail` guard'ları, bölge senkronizasyonu hiç test edilmiyor.

### Eksiklik 7: `@always` Router State Testleri — YOK

**Problem:** `idleStateResolver`, `farmerStateResolver`, `findeks.reportSaved` gibi `@always` state'ler guard'lara göre anında yönlendirme yapar. Bu routing logic test edilmiyor.

## Gerçek Dünya Test İhtiyaçları (Pattern'lerden Çıkarılan)

### İhtiyaç 1: `withContext()` — DB-Backed Context Initialization
Mevcut pattern:
```php
$machine = Machine::create();
$machine->state->context->set('application', $application);
```
İhtiyaç: TestMachine'in model nesneleri ve computed property'lerle context başlatabilmesi. `Machine::test()` context override'ı post-start uyguluyor — bu Application gibi entry action'lı makinelerde sorun.

### İhtiyaç 2: `faking()` + Orchestration Pattern
Mevcut pattern:
```php
SubAction1::shouldRun()->once();
SubAction2::shouldRun()->never();
OrchestratorAction::run($context, $event);
```
İhtiyaç: TestMachine seviyesinde bir transition'ın hangi alt-action'ları çağırdığını doğrulama.

### İhtiyaç 3: Inline Behavior Testing
Mevcut pattern: `Machine::getGuard('inlineGuardName')` ile closure çıkarma
İhtiyaç: TestMachine'in inline behavior'ları (closure) kolayca test edilebilir kılması. `getGuard/getAction/getCalculator` zaten var ama TestMachine ile entegre değil.

### İhtiyaç 4: Parametric Guard Testing
Mevcut pattern:
```php
$guard = Machine::getGuard('checkDaysAfterCompletionGuard');
$guard($context, ['7']);  // ikinci parametre = guard argümanları
```
İhtiyaç: Parametrik guard'lar (`guardName:param1,param2`) için test desteği. `runWithState()` bunu `$arguments` parametresiyle destekliyor ama dokümante edilmemiş.

### İhtiyaç 5: `assertNotificationSentTo` Integration
Her action testinde `Notification::fake()` + `Notification::assertSentTo()` tekrarlanıyor. Machine-level testte bu otomatik olabilir.

### İhtiyaç 6: DB State Verification After Transition
```php
->send('APPROVE')
->assertState('approved')
->assertDatabaseHas(Application::class, ['status' => 'approved'])
```
Persistence test'lerle machine transition test'lerini birleştirme ihtiyacı.

### İhtiyaç 7: Scenario-Aware Testing
```php
ApplicationMachine::test()
    ->withScenario('ml_score_approved_scenario')
    ->send(ProtocolStartedEvent::forTesting([...]))
    ->assertState('approved.idle');
```

### İhtiyaç 8: Event Payload Factory Integration
Backend `EventFactory` pattern'i kullanıyor:
```php
$event = ApplicationStartedEventFactory::new()->direct()->make();
```
TestMachine'in bu factory'lerle doğal çalışması lazım — `send()` zaten `EventBehavior` kabul ediyor (task 1'de düzelttik).

## TestMachine API Gap Analizi

| Gerçek Dünya İhtiyacı | Mevcut API | Eksik |
|----------------------|-----------|-------|
| End-to-end state flow | `send()` + `assertState()` | `withoutPersistence()` var ✓ |
| Guard sıralama | `assertGuarded()` | Sadece "guarded or not" — hangi guard fail etti bilgisi yok |
| Inline behavior test | `getGuard/getAction` (ResolvesBehaviors) | TestMachine entegrasyonu yok |
| DB-backed context | `Machine::test(['key' => $model])` | Post-start override — entry action'lar default context görür |
| Scenario testing | Yok | `withScenario()` lazım |
| Notification assertions | Yok | Laravel `Notification::fake()` ile birlikte kullanım belgelenmeli |
| DB assertions after transition | Yok | `assertDatabaseHas` ile chain lazım |
| Parametric guard args | `runWithState($state, null, ['7'])` | Dokümante edilmemiş |
| Event factory integration | `send(EventBehavior)` | Zaten var ✓ (task 1) |
| assertResult | `assertResult()` | Var ✓ (task 12) |
| Parallel state flow | `assertRegionState()` | Var ama CarSalesMachine'de test edilmedi |
| `@always` router testing | Yok | Otomatik geçiş test desteği lazım |
| `@done/@fail` testing | Yok | Parallel completion/failure assertions lazım |
