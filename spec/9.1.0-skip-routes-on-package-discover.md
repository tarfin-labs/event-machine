# Skip Machine Route Registration During `package:discover`

**Status:** Draft
**Date:** 2026-03-30

---

## Problem

`MachineRouter::register()` calls `$machineClass::definition()` at boot time to discover endpoint metadata for route registration. The `definition()` call builds the entire state tree, parses transitions, validates config, and resolves forwarded endpoints (including child machine definitions). Any failure in this chain crashes `composer update` because `package:discover` boots all service providers, which loads route files.

```
composer update
  → post-autoload-dump: php artisan package:discover
    → service providers boot → route files load
      → MachineRouter::register(SomeMachine::class)
        → $machineClass::definition() → full construction + validation → throws
          → composer update fails
```

`package:discover` is triggered **automatically** by composer — the developer has no control over it. Unlike `migrate` or `route:list` (which the developer runs intentionally and can fix config errors before retrying), a broken `package:discover` blocks `composer update` itself, creating a chicken-and-egg: the developer may need the update to fix the config.

---

## Solution

Skip route registration during `package:discover`. This command only registers service providers — it does not need machine routes.

Laravel provides a first-class API for this: `app()->runningConsoleCommand()` (Laravel 10+). This is the same pattern used by **Laravel Telescope** to skip heavy boot work during infrastructure commands.

**File:** `src/Routing/MachineRouter.php`

```php
public static function register(string $machineClass, array $options): void
{
    if (app()->runningConsoleCommand('package:discover')) {
        return;
    }

    $definition = $machineClass::definition();
    // ... rest unchanged
}
```

### `MachineDiscovery` — Already Safe

`MachineServiceProvider::registerTimerSweeps()` calls `MachineDiscovery::findTimerMachines()` which already wraps `definition()` in `catch (\Throwable) { continue; }`. No change needed.

---

## Testing Plan

1. **`package:discover` skips route registration**
   - Set `$_SERVER['argv'] = ['artisan', 'package:discover']` + console environment
   - Call `MachineRouter::register()`
   - Assert no routes registered

2. **Normal boot registers routes**
   - HTTP context (not console)
   - Call `MachineRouter::register()` — assert routes are registered

3. **Other artisan commands register routes normally**
   - Set `$_SERVER['argv'] = ['artisan', 'route:list']`
   - Call `MachineRouter::register()` — assert routes are registered

## Risk Assessment

| Risk | Likelihood | Mitigation |
|------|-----------|------------|
| Future composer script needs routes | Very low | `package:discover` has a fixed scope (service provider registration) |
| `runningConsoleCommand()` API removed | Very low | Core Laravel API used by Telescope and other first-party packages |
