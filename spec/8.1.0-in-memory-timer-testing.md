# In-Memory Timer Testing for TestMachine::define()

## Problem

`advanceTimers()` requires persistence (`shouldPersist = true`) because it:
1. Calls `$this->machine->persist()` to sync `machine_current_states`
2. Backdates `state_entered_at` in the database
3. Backdates `last_fired_at` in `machine_timer_fires`
4. Queries `MachineCurrentState::forInstance()` during sweep
5. Records fire history in `machine_timer_fires`

This means `TestMachine::define()` and `withContext()` (both `shouldPersist = false`) cannot use `advanceTimers()`. Users must fall back to manual `send('TIMER_EVENT')` which loses the duration semantics.

## Current assertHasTimer()

Already exists at `TestMachine.php:945` — checks definition-level timer config, works without persistence. But it only asserts "a timer exists", not "the timer would fire after X seconds".

## Goal

Enable `advanceTimers(Timer $duration)` to work without database persistence by tracking timer state in-memory.

## Design

### New In-Memory State

Add properties to `TestMachine`:

```php
/** @var \Carbon\Carbon|null When the current state was entered (in-memory mode) */
private ?\Carbon\Carbon $inMemoryStateEnteredAt = null;

/** @var array<string, array{last_fired_at: \Carbon\Carbon, fire_count: int, status: string}> */
private array $inMemoryTimerFires = [];

/** @var string|null Last tracked state ID for detecting transitions */
private ?string $lastTrackedStateId = null;
```

### State Entry Tracking

When `shouldPersist === false`, record `now()` whenever the machine transitions. A single `trackStateEntry()` method handles this — call it at the end of `send()`, `simulateChildDone()`, `simulateChildFail()`, `simulateChildTimeout()`, and in all factory constructors (`define()`, `withContext()`, `for()`) for the initial state.

When the state changes, old-state **active** timer fires are cleared (recurring timers stop). Historical records (`fired`, `exhausted`) are preserved for assertions — matching DB behavior where `machine_timer_fires` records persist independently of `machine_current_states`.

```php
private function trackStateEntry(): void
{
    $currentId = $this->machine->state->currentStateDefinition->id;

    if ($currentId !== $this->lastTrackedStateId) {
        $this->inMemoryStateEnteredAt = now();
        $this->lastTrackedStateId     = $currentId;

        // Keep: all fires for current state (any status)
        // Keep: historical fires for old states (fired/exhausted) — for assertions
        // Remove: active recurring fires for old states — stop @every timers
        $this->inMemoryTimerFires = array_filter(
            $this->inMemoryTimerFires,
            fn (array $fire, string $key): bool =>
                str_starts_with($key, $currentId.':')  // current state → keep all
                || $fire['status'] !== 'active',         // old state → keep historical (fired/exhausted)
            ARRAY_FILTER_USE_BOTH,
        );
    }
}
```

### Modified Callers

`send()`, `simulateChildDone()`, `simulateChildFail()`, and `simulateChildTimeout()` need a `trackStateEntry()` call appended. `send()` may have multiple return paths (early return on validation failure, guarded transition, etc.) — ensure `trackStateEntry()` runs on ALL paths that could change state, typically as the last statement before the final `return $this`:

```php
// In send() — after existing logic, before return
public function send(...): self
{
    // ... existing logic ...

    // Track state entry for in-memory timer support
    if ($this->machine->definition->shouldPersist === false) {
        $this->trackStateEntry();
    }

    return $this;
}
```

Same pattern for `simulateChildDone()`, `simulateChildFail()`, `simulateChildTimeout()`.

All factory methods that produce non-persisted machines must initialize state tracking:

```php
// define()
public static function define(array $config, array $behavior = []): self
{
    // ... existing logic ...
    $instance = new self($machine);
    $instance->trackStateEntry(); // record initial state entry time

    return $instance;
}

// withContext()
public static function withContext(string $machineClass, array $context): self
{
    // ... existing logic ...
    $instance = new self($machine);
    $instance->trackStateEntry(); // record initial state entry time

    return $instance;
}

// for() — machine may or may not have persistence
public static function for(Machine $machine): self
{
    $instance = new self($machine);

    if ($machine->definition->shouldPersist === false) {
        $instance->trackStateEntry();
    }

    return $instance;
}
```

### New advanceTimersInMemory()

Uses `->copy()->subSeconds()` to avoid Carbon mutability issues — `Carbon\Carbon` is mutable, so `subSeconds()` mutates the instance in-place. Without `copy()`, any other reference to the same Carbon instance would see the mutation.

```php
private function advanceTimersInMemory(Timer $duration): self
{
    if ($this->inMemoryStateEnteredAt === null) {
        return $this;
    }

    // Simulate time passing (copy() avoids Carbon mutability issues)
    $this->inMemoryStateEnteredAt = $this->inMemoryStateEnteredAt
        ->copy()->subSeconds($duration->inSeconds());

    // Also backdate in-memory timer fires for @every
    foreach ($this->inMemoryTimerFires as $key => &$fire) {
        if ($fire['status'] === 'active') {
            $fire['last_fired_at'] = $fire['last_fired_at']
                ->copy()->subSeconds($duration->inSeconds());
        }
    }
    unset($fire);

    // Run sweep in-memory
    $this->processTimersInMemory();

    return $this;
}
```

### New processTimersInMemory()

Accesses the current state's transitions directly instead of iterating all states in `idMap`. This is simpler and avoids a subtle issue: if a timer fires and causes a state transition mid-loop, iterating `idMap` with a live `currentStateDefinition` check would see the new state's timers too. Direct access via a local variable snapshots the current state before processing — `$currentState` holds the reference captured before any timer fires, and PHP's `foreach` iterates a copy of the array, so mid-loop state changes don't affect the iteration.

```php
private function processTimersInMemory(): self
{
    $currentState = $this->machine->state->currentStateDefinition;

    foreach ($currentState->transitionDefinitions ?? [] as $transitionDef) {
        if ($transitionDef->timerDefinition === null) {
            continue;
        }

        $timer = $transitionDef->timerDefinition;

        if ($timer->isAfter()) {
            $this->processAfterTimerInMemory($timer);
        } elseif ($timer->isEvery()) {
            $this->processEveryTimerInMemory($timer);
        }
    }

    return $this;
}
```

### After Timer (In-Memory)

```php
private function processAfterTimerInMemory(TimerDefinition $timer): void
{
    $deadline = now()->subSeconds($timer->delaySeconds);

    if ($this->inMemoryStateEnteredAt->greaterThan($deadline)) {
        return; // Not past deadline
    }

    $key = $timer->key();

    // Check dedup — already fired?
    if (isset($this->inMemoryTimerFires[$key])
        && $this->inMemoryTimerFires[$key]['status'] === 'fired') {
        return;
    }

    // Send event
    $this->machine->send(['type' => $timer->eventName]);

    // Record fire (persists across state transitions for assertions)
    $this->inMemoryTimerFires[$key] = [
        'last_fired_at' => now(),
        'fire_count'    => 1,
        'status'        => 'fired',
    ];

    // Track state entry if transition happened
    $this->trackStateEntry();
}
```

### Every Timer (In-Memory)

```php
private function processEveryTimerInMemory(TimerDefinition $timer): void
{
    $key      = $timer->key();
    $fireData = $this->inMemoryTimerFires[$key] ?? null;

    // Check exhausted
    if ($fireData !== null && $fireData['status'] === 'exhausted') {
        return;
    }

    $lastFiredAt  = $fireData['last_fired_at'] ?? $this->inMemoryStateEnteredAt;
    $currentCount = $fireData['fire_count'] ?? 0;

    if (now()->diffInSeconds($lastFiredAt, absolute: true) < $timer->delaySeconds) {
        return;
    }

    // Check max/then
    if ($timer->max !== null && $currentCount >= $timer->max) {
        if ($timer->then !== null) {
            $this->machine->send(['type' => $timer->then]);
            $this->trackStateEntry();
        }

        $this->inMemoryTimerFires[$key] = [
            'last_fired_at' => now(),
            'fire_count'    => $currentCount,
            'status'        => 'exhausted',
        ];

        return;
    }

    // Send event
    $this->machine->send(['type' => $timer->eventName]);

    // Track fire
    $this->inMemoryTimerFires[$key] = [
        'last_fired_at' => now(),
        'fire_count'    => $currentCount + 1,
        'status'        => 'active',
    ];

    $this->trackStateEntry();
}
```

Add in-memory fire assertions:

```php
public function assertTimerFired(string $eventName): self
{
    if ($this->machine->definition->shouldPersist === false) {
        return $this->assertTimerFiredInMemory($eventName);
    }
    // ... existing DB-based logic ...
}

private function assertTimerFiredInMemory(string $eventName): self
{
    $found = false;

    foreach ($this->inMemoryTimerFires as $key => $fire) {
        if (str_contains($key, ":{$eventName}:")) {
            $found = true;
            break;
        }
    }

    if (!$found) {
        throw new AssertionFailedError(
            "Expected timer event '{$eventName}' to have been fired (in-memory), but no record found."
        );
    }

    return $this;
}
```

Same pattern for `assertTimerNotFired()`.

## Test Scenarios

1. **Basic after timer with define()**: Define a machine with `after: Timer::seconds(30)`, advance by 30s, assert state changed
2. **After timer not yet due**: Advance by 20s (< 30s deadline), assert state unchanged
3. **After timer dedup**: Advance twice past deadline, assert timer fires only once
4. **Every timer fires**: Define `every: Timer::minutes(1)`, advance by 60s, assert event sent
5. **Every timer with max**: `max: 3`, advance 3x, assert exactly 3 fires then exhausted
6. **Every timer with then**: `max: 2, then: 'TIMEOUT'`, advance past max, assert TIMEOUT sent
7. **State transition resets after timers**: After timer fires and transitions, new state's timers start fresh
8. **assertHasTimer with duration**: `assertHasTimer('EXPIRE', Timer::seconds(120))` passes/fails correctly
9. **assertTimerFired in-memory**: Works without persistence
10. **assertTimerNotFired in-memory**: Works without persistence
11. **Mixed: create() still uses DB path**: Existing DB-based tests remain unaffected
12. **Guard blocks timer transition**: Timer fires but guard prevents state change — state stays, fire is recorded
13. **withContext() + advanceTimers()**: Machine created via `SomeMachine::withContext([...])` supports `advanceTimers()` (also `shouldPersist = false`)
14. **Cumulative advanceTimers()**: `advanceTimers(Timer::seconds(60))` then `advanceTimers(Timer::seconds(61))` — total 121s exceeds 120s deadline
15. **assertTimerFired survives state transition**: After timer fires and machine transitions, `assertTimerFired()` still finds the historical record (fire history not lost on state change)
16. **for() with non-persisted machine**: `TestMachine::for($nonPersistedMachine)->advanceTimers(...)` works (initial state entry tracked)

## Backward Compatibility

- `advanceTimers()` with persistence = existing DB logic (no change)
- `advanceTimers()` without persistence = new in-memory logic (was RuntimeException before)
- `assertHasTimer()` with optional duration = additive, existing calls unaffected
- `assertTimerFired/NotFired` auto-detect mode = transparent to user

## Edge Cases

- **Carbon::setTestNow()**: Users may freeze time in tests. In-memory logic respects `now()` which honors test time.
- **Carbon mutability**: All time manipulation uses `->copy()->subSeconds()` to avoid mutating shared Carbon instances.
- **Multiple timers on same state**: Each has unique `key()` (`stateId:eventName:delaySeconds`), handled independently.
- **Timer on initial state**: `trackStateEntry()` called in factory sets initial timestamp.
- **Guard blocks timer transition**: If `send()` is guarded and state doesn't change, fire still records (matches DB behavior where the command fires regardless).
- **Timer fires causing state transition**: `processTimersInMemory()` snapshots the current state's transitions before iterating, so newly-entered state's timers are not processed in the same sweep.
- **Fire history survives state transitions**: `trackStateEntry()` only removes `active` (recurring @every) entries for old states. Historical `fired` and `exhausted` records are preserved, so `assertTimerFired()` works even after the timer caused a state transition. This matches DB behavior where `machine_timer_fires` persists independently of `machine_current_states`.
- **send() multi-return paths**: `trackStateEntry()` must run on all return paths that could change state. During implementation, verify all `return $this` statements in `send()` are preceded by the conditional `trackStateEntry()` call, or consolidate to a single return point.
